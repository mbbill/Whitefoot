# Independent ResourceProfile source route

This directory is the first source-to-role prerequisite for ResourceProfile
v1 measurement. It is evidence code, not compiler code and not language
authority. It selects no numerical maximum.

The route reads an ordered nonempty PROG-2 bundle, independently checks its
portable logical paths, lexes and parses each exact canonical successor source
with the previously independent handwritten FORM-2 parser family, constructs
one flattened `Program` topology, derives production-child `NodePath`s,
projects the complete owner-approved role and scope matrices, runs FN-8 before
role classification, and selects inventory before lexical-resolution issues.
It computes every source/tree/role/scope count that is a property of those
products, including all checked cross-field receipts and the candidate charged
tree-byte total.

The parser reuse is narrow and fail closed. `identities.py` pins every imported
parser file and rejects source references to compiler crates, proposal
diagnostic models, either measurement-route directory, or the archive. The
route imports no production compiler crate, no workload producer module, no
proposal `diagnostic_evidence`, and no other evidence route. Its neutral
manifest decoder is an independent implementation and accepts only producer
revision `4ecae9410fb82b62c2e8da595d944d29b1de8ae9f9c57983c1eaa393a1bc07d3`.

Six fields are deliberately not available yet: 9, 14, 15, 16, 17, and 33.
They measure algorithm executions rather than source/tree cardinalities. Every
receipt marks them `trace-required`; `TRACE-GAPS.md` defines the independent
schedule replay needed to close them. The receipt status therefore remains
`trace-incomplete` and cannot enter a numerical owner-review packet.

Run the focused tests with:

```text
python3 -m unittest discover -s optimizer-language-research/implementation/phase5-resource-profile/source-route -p 'test_*.py'
```

Run one receipt with matching repeated logical/source arguments:

```text
python3 optimizer-language-research/implementation/phase5-resource-profile/source-route/run.py \
  --logical-path demand/compiler-000001.wf \
  --source-file /tmp/compiler.wf \
  --manifest /tmp/compiler.json \
  --output /tmp/source-route-receipt.json
```

Operational CLI read caps protect the evidence process; they are not proposed
ResourceProfile values. An FN-8 rejection correctly leaves the ordinary
resolution-count fields `not-derived`, because R-04 forbids that counting
subpass. No resource or capability claim is inferred from a receipt.
