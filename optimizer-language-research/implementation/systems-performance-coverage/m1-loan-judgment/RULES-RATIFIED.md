# M1 Loan/Freeze Judgment — Ratified Rules

Status: RATIFIED consolidation per D18. This document supersedes the pre-review
rule text in `M1-PAPER-RESULT.md` (§ "The 15 rules"). The original is kept
UNTOUCHED as the audit trail; this file is the current normative rule set. It
folds in the four ratified amendments and the declaration-time tightening from
`AMENDMENTS.md`; every amendment is marked inline `[AMD-n]` and cross-referenced
at the end. Rule text not carrying an `[AMD-n]` mark is verbatim from the
original. The machine spec (`machine_spec.md`) carries the operational detail;
no rule hides a second mechanism.

## The 15 rules (ratified)

1. R1 CONF-1 (Confined formation and kind). A struct or enum type may be declared `confined(uniq)` or `confined(shr)`; it is affine, carries at least one region parameter if it can hold a loan, and its fields may be borrows (&'p T or &uniq 'p T at declared region parameters), confined types instantiated at declared region parameters, or owned non-confined types. Only confined types may contain borrow-typed or confined-typed fields; a non-confined type containing either is ill-formed, and instantiating any generic or sealed-form type parameter with a confined type is ill-formed (checked after monomorphization). A confined(shr) type must not transitively contain any &uniq field or any confined(uniq) field. Sealed-form token, cursor, guard, and endpoint result types are opaque confined types declared in the form table; user code cannot construct opaque confined values.

2. R2 CONF-2 (Stack confinement). A confined value may exist only as a local let binding, a rebind target, a call argument (by own or by a statement-scoped minted borrow), a call result, a match scrutinee, a match-arm token binding (R11), or a field of another confined value under construction. A statement that would store a confined value into heap data, a sealed-form container, a non-confined aggregate, or a global is rejected at that statement.

3. R3 CONF-3 (Affinity, minting, projection). A confined binding is affine: passing it by own, destructuring it (R11), and scope end each consume it, and any use after consumption is rejected. A live confined binding may mint one statement-scoped borrow (& or &uniq) per statement, which the receiving call consumes (T-A) and which transfers no loans. Field projection through a live confined binding, or through a borrow of one, yields only a statement-scoped borrow consumed by the receiving call; no multi-statement binding is ever created from a confined value's field except by R11 destructure.

4. R4 LOAN-1 (Loan state). The checker keeps one loan table per function: a finite set of entries (P, k, h), where P is a resolved place (OWN-5: a root binding plus a possibly empty field path), k in {uniq, shr} is fixed as the declared kind of the holder's confined type, and h is the binding currently holding the token; each holder also records, per region parameter of its type, its source place. A statement touching place Q is governed by every entry whose place overlaps Q (either place a prefix of the other, same root). Entries change only at issue (R5), consume and scope end (R7), and holder transfer (R8, R11); no other analysis reads or writes this state.

5. R5 LOAN-2 (Issue). An operation or construction issuing a loan of kind k on source place P requires: if k = uniq, no entry overlapping P; if k = shr, no uniq entry overlapping P; and the access to P minted in that statement must itself be legal under R6. The new holder is bound by a fresh let (or rebind), its region parameters instantiate from P (a reference binding's declared region, else the block region of P's root), and (P, k, holder) is added at statement exit. A user confined construction must instantiate every region parameter of its type from a captured borrow mint or a moved confined field in that same construction — leaving any region parameter unbound is rejected — and it issues one entry per captured borrow's source place with the constructed type's kind. An issuing form op whose receiver is itself a live holder issues on that holder's recorded source place (a sibling loan); **[AMD-5] the `issues K on source(receiver)` clause is legal only for shr-from-shr — a result kind or receiver kind of uniq is rejected at declaration (R15), never fires, and is unwritable here.**

6. R6 LOAN-3 (Freeze). While any entry on P is live, P and every place overlapping it cannot be moved, rebound, assigned over, passed by own, explicitly dropped, or reached by a compiler-derived drop (R7). While a uniq entry (P, uniq, h) is live, no operation on, or borrow mint from, P is permitted; while only shr entries are live on P, & mints, &-receiver operations, and further shr issues on P are permitted, and &uniq mints, &uniq- or own-receiver operations, and uniq issues are rejected. Sole exception in both regimes: a single statement that consumes or reissues holder h may mint the borrows of P that the callee's signature declares, provided h's entries are the only entries on P. **[AMD-2] Mode-capability clause: a &uniq mint (or own-pass) of a place whose root binding was borrowed &-shared is rejected; only an owned local or a &uniq-borrowed source can be uniquely accessed. Mint legality no longer consults the loan table alone, closing M1's silent dependence on the base ownership (OWN) layer by which a &-shared parameter could be &uniq-minted inside a body (BREAK-1).**

7. R7 LOAN-4 (Consume and scope end). A call that consumes holder h — declared `consumes`/`reissues` in the form table or derived per R10 — removes every entry held by h after the statement's legality checks and before its issues, so consume-and-reissue composes in one statement. At scope end, bindings are processed in reverse binding order: a live holder is auto-consumed (its compiler-derived drop performs the form-declared release — unlock, close, retire — and removes its entries), and any owned binding still overlapped by a live entry when its derived drop is reached is a checked error. Entries seeded at function entry (R10c) expire at function exit without release.

8. R8 LOAN-5 (Holder transfer). Moving a live holder — by plain let-move, by `rebind h = h2` (which requires identical confined type and a consumed destination), or into a field of a confined construction — renames its entries' holder to the destination binding (for constructions, to the constructed binding), merges its source records, and consumes the source binding; a loan's kind never changes in transfer, and no operation duplicates a holder or an entry.

9. R9 SIG-1 (Signature ties). In every function signature, each region parameter 'p of each confined parameter or confined result must tie to exactly one borrow-mode parameter whose declared region is 'p; if no distinct borrow-mode parameter carries 'p and the confined parameter is itself passed by borrow, it ties to itself (self-tie); zero candidates, multiple candidates, or a self-tie needed for an own-mode parameter or a result reject the declaration. Form-table ops declare their loan clause (issues K / issues K on source(receiver) / consumes / reissues / none) in the same vocabulary and are subject to the same call-site checks as derived signatures. **[AMD-3] Form-op receiver-holder tie (replacing the undocumented `is_form => SELF` carve-out): a form-table op's confined receiver region with no distinct borrow candidate ties to the receiver holder itself; an `issues K on source(receiver)` clause issues the result on that receiver holder's recorded source. The receiver-holder tie skips the distinct-place R10(a) comparison but STILL applies an identity brand — the token must hold a live entry at its recorded source (a genuine issued token); a local confined value with no recorded source reaching a form op is rejected as forged, and a self-tied parameter is trusted from the signature. This makes "form ops are subject to the same call-site checks as derived signatures" literally true and restores the R10(a) coverage the carve-out silently dropped.**

10. R10 SIG-2 (Signature effects: caller and callee). At every call: (a) each confined argument must hold, for each distinct-tied region parameter, a live entry whose source place is exactly the resolved place of the argument bound to that tie — a missing entry or any mismatch rejects (this is the instance-brand check; self-ties skip it and a self-tied-origin value cannot be passed into a distinct-tie slot); (b) own-mode confined arguments are consumed (R7), and each confined result issues, per region parameter, an entry of the result type's kind on the tied argument's place, held by the freshly bound result, with R5's preconditions checked at the call; (c) every body is checked with its loan table seeded from its own signature — one entry (tie binding, kind, h) per distinct-tied confined parameter h — and at every return, after auto-consuming non-returned locals, the live entries on tied parameters must equal exactly the signature-implied caller effect. The caller trusts only the signature.

11. R11 MATCH-1 (Confined destructure). A match may destructure a live confined scrutinee only if it is an enum whose every variant packs all of its borrow-typed and confined-typed payload into exactly one confined field of the same kind and region parameters as the scrutinee's type; the match consumes the scrutinee, and in each arm the scrutinee's entries rename to that arm's bound token. Any other destructure of a confined value — any struct pattern, or any variant with zero or several such fields — is rejected; all other field access remains statement-scoped per R3.

12. R12 JOIN-1 (Match, loop, break, return agreement). Every match arm is checked from a copy of the entry state; **[AMD-4] states are compared as the loan-table entries (INCLUDING holder names) and the identical liveness of ALL bindings declared outside the construct, taken after the arm's or body's own scope-end processing (R7)** (the ratified comparison domain replaces the original "holder liveness" phrase, aligning the rule text with the machine spec and checker). All fall-through arms must agree exactly, including holder names; all break states of a loop must agree pairwise and become the post-loop state; a loop body's back-edge state must equal its entry state unless the back edge is syntactically unreachable (every path ends in break or return — decidable by inspection in match-only ANF). Any disagreement rejects; states are never merged by approximation.

13. R13 ESC-1 (Region escape). A confined value may be returned only if every region parameter of its type is a region parameter of the enclosing function; returning a confined value carrying any locally introduced region is rejected. Because every loanable confined type carries a region parameter (R1), every holder's region is its source's region (R5), and regions are lexical, a holder's scope always lies within its source's scope, and R7's reverse-order scope processing reaches every holder before its source.

14. R14 PAR-1 (Scoped par boundary). Par bodies are named functions; the par form's signature marks each argument slot replicate-shared or split-unique. A replicate-shared slot accepts a & mint of a source place with no uniq entries, or a & mint of a live holder all of whose held entries are shr; a holder holding any uniq entry is rejected there. A split-unique slot accepts one &uniq mint (legal under R6) that the trusted form divides into disjoint per-body views for the call's extent. All mints are consumed by the par statement, which returns only after joining every body, so every captured region lexically encloses the join; each body's effect row must match its slot modes (no writes through replicated slots; writes only into the body's own split view). **[AMD-1] Statement-local mint-disjointness: every mint of ONE statement is accumulated as a pseudo-entry (split / &uniq -> uniq; replicate / & -> shr); any two mints on overlapping places where at least one is unique are rejected, while replicate+replicate (shr+shr) on the same place stays legal. This is the sole authority for par cross-slot disjointness AND closes same-statement aliased &uniq mints in ordinary calls (E2d / S1a) — previously each slot was checked only against the pre-existing loan table, so two slots never saw each other.**

15. R15 FORM-1 (Fail-closed form table). Every operation of a loanable sealed form carries exactly one loan clause; a call whose loan behavior is neither declared in the form table nor derived by R9/R10 is rejected — absence of declaration is rejection, never default-accept. An op may be declared with & receiver mode on a loanable form only if its internals both preserve every interior address and are safe under concurrent invocation; these obligations, the release actions run by holder drops, and issues-on-source(receiver) clauses are sealed-form proof obligations subject to hostile fact-channel review before shipping — a green checker is not that review. **[AMD-5] Declaration-time tightening (D18): the form-table pre-pass fails closed on provably-dead `issues K on source(receiver)` declarations — an op whose result kind is uniq or whose receiver kind is uniq is rejected AT DECLARATION, because the receiver's own live entry always blocks a sibling uniq issue so the op can never legally fire. The clause is thereby restricted to shr-from-shr, the shr-from-shr-only restriction written into the spec at the same time as required by the amendment's condition.**
