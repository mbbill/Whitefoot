# M1 merged rules and machine spec

## Rules (15)

1. R1 CONF-1 (Confined formation and kind). A struct or enum type may be declared `confined(uniq)` or `confined(shr)`; it is affine, carries at least one region parameter if it can hold a loan, and its fields may be borrows (&'p T or &uniq 'p T at declared region parameters), confined types instantiated at declared region parameters, or owned non-confined types. Only confined types may contain borrow-typed or confined-typed fields; a non-confined type containing either is ill-formed, and instantiating any generic or sealed-form type parameter with a confined type is ill-formed (checked after monomorphization). A confined(shr) type must not transitively contain any &uniq field or any confined(uniq) field. Sealed-form token, cursor, guard, and endpoint result types are opaque confined types declared in the form table; user code cannot construct opaque confined values.
2. R2 CONF-2 (Stack confinement). A confined value may exist only as a local let binding, a rebind target, a call argument (by own or by a statement-scoped minted borrow), a call result, a match scrutinee, a match-arm token binding (R11), or a field of another confined value under construction. A statement that would store a confined value into heap data, a sealed-form container, a non-confined aggregate, or a global is rejected at that statement.
3. R3 CONF-3 (Affinity, minting, projection). A confined binding is affine: passing it by own, destructuring it (R11), and scope end each consume it, and any use after consumption is rejected. A live confined binding may mint one statement-scoped borrow (& or &uniq) per statement, which the receiving call consumes (T-A) and which transfers no loans. Field projection through a live confined binding, or through a borrow of one, yields only a statement-scoped borrow consumed by the receiving call; no multi-statement binding is ever created from a confined value's field except by R11 destructure.
4. R4 LOAN-1 (Loan state). The checker keeps one loan table per function: a finite set of entries (P, k, h), where P is a resolved place (OWN-5: a root binding plus a possibly empty field path), k in {uniq, shr} is fixed as the declared kind of the holder's confined type, and h is the binding currently holding the token; each holder also records, per region parameter of its type, its source place. A statement touching place Q is governed by every entry whose place overlaps Q (either place a prefix of the other, same root). Entries change only at issue (R5), consume and scope end (R7), and holder transfer (R8, R11); no other analysis reads or writes this state.
5. R5 LOAN-2 (Issue). An operation or construction issuing a loan of kind k on source place P requires: if k = uniq, no entry overlapping P; if k = shr, no uniq entry overlapping P; and the access to P minted in that statement must itself be legal under R6. The new holder is bound by a fresh let (or rebind), its region parameters instantiate from P (a reference binding's declared region, else the block region of P's root), and (P, k, holder) is added at statement exit. A user confined construction must instantiate every region parameter of its type from a captured borrow mint or a moved confined field in that same construction — leaving any region parameter unbound is rejected — and it issues one entry per captured borrow's source place with the constructed type's kind. An issuing form op whose receiver is itself a live holder issues on that holder's recorded source place (a sibling loan).
6. R6 LOAN-3 (Freeze). While any entry on P is live, P and every place overlapping it cannot be moved, rebound, assigned over, passed by own, explicitly dropped, or reached by a compiler-derived drop (R7). While a uniq entry (P, uniq, h) is live, no operation on, or borrow mint from, P is permitted; while only shr entries are live on P, & mints, &-receiver operations, and further shr issues on P are permitted, and &uniq mints, &uniq- or own-receiver operations, and uniq issues are rejected. Sole exception in both regimes: a single statement that consumes or reissues holder h may mint the borrows of P that the callee's signature declares, provided h's entries are the only entries on P.
7. R7 LOAN-4 (Consume and scope end). A call that consumes holder h — declared `consumes`/`reissues` in the form table or derived per R10 — removes every entry held by h after the statement's legality checks and before its issues, so consume-and-reissue composes in one statement. At scope end, bindings are processed in reverse binding order: a live holder is auto-consumed (its compiler-derived drop performs the form-declared release — unlock, close, retire — and removes its entries), and any owned binding still overlapped by a live entry when its derived drop is reached is a checked error. Entries seeded at function entry (R10c) expire at function exit without release.
8. R8 LOAN-5 (Holder transfer). Moving a live holder — by plain let-move, by `rebind h = h2` (which requires identical confined type and a consumed destination), or into a field of a confined construction — renames its entries' holder to the destination binding (for constructions, to the constructed binding), merges its source records, and consumes the source binding; a loan's kind never changes in transfer, and no operation duplicates a holder or an entry.
9. R9 SIG-1 (Signature ties). In every function signature, each region parameter 'p of each confined parameter or confined result must tie to exactly one borrow-mode parameter whose declared region is 'p; if no distinct borrow-mode parameter carries 'p and the confined parameter is itself passed by borrow, it ties to itself (self-tie); zero candidates, multiple candidates, or a self-tie needed for an own-mode parameter or a result reject the declaration. Form-table ops declare their loan clause (issues K / issues K on source(receiver) / consumes / reissues / none) in the same vocabulary and are subject to the same call-site checks as derived signatures.
10. R10 SIG-2 (Signature effects: caller and callee). At every call: (a) each confined argument must hold, for each distinct-tied region parameter, a live entry whose source place is exactly the resolved place of the argument bound to that tie — a missing entry or any mismatch rejects (this is the instance-brand check; self-ties skip it and a self-tied-origin value cannot be passed into a distinct-tie slot); (b) own-mode confined arguments are consumed (R7), and each confined result issues, per region parameter, an entry of the result type's kind on the tied argument's place, held by the freshly bound result, with R5's preconditions checked at the call; (c) every body is checked with its loan table seeded from its own signature — one entry (tie binding, kind, h) per distinct-tied confined parameter h — and at every return, after auto-consuming non-returned locals, the live entries on tied parameters must equal exactly the signature-implied caller effect. The caller trusts only the signature.
11. R11 MATCH-1 (Confined destructure). A match may destructure a live confined scrutinee only if it is an enum whose every variant packs all of its borrow-typed and confined-typed payload into exactly one confined field of the same kind and region parameters as the scrutinee's type; the match consumes the scrutinee, and in each arm the scrutinee's entries rename to that arm's bound token. Any other destructure of a confined value — any struct pattern, or any variant with zero or several such fields — is rejected; all other field access remains statement-scoped per R3.
12. R12 JOIN-1 (Match, loop, break, return agreement). Every match arm is checked from a copy of the entry state; states are compared as the loan table and holder liveness restricted to bindings declared outside the construct, taken after the arm's or body's own scope-end processing (R7). All fall-through arms must agree exactly, including holder names; all break states of a loop must agree pairwise and become the post-loop state; a loop body's back-edge state must equal its entry state unless the back edge is syntactically unreachable (every path ends in break or return — decidable by inspection in match-only ANF). Any disagreement rejects; states are never merged by approximation.
13. R13 ESC-1 (Region escape). A confined value may be returned only if every region parameter of its type is a region parameter of the enclosing function; returning a confined value carrying any locally introduced region is rejected. Because every loanable confined type carries a region parameter (R1), every holder's region is its source's region (R5), and regions are lexical, a holder's scope always lies within its source's scope, and R7's reverse-order scope processing reaches every holder before its source.
14. R14 PAR-1 (Scoped par boundary). Par bodies are named functions; the par form's signature marks each argument slot replicate-shared or split-unique. A replicate-shared slot accepts a & mint of a source place with no uniq entries, or a & mint of a live holder all of whose held entries are shr; a holder holding any uniq entry is rejected there. A split-unique slot accepts one &uniq mint (legal under R6) that the trusted form divides into disjoint per-body views for the call's extent. All mints are consumed by the par statement, which returns only after joining every body, so every captured region lexically encloses the join; each body's effect row must match its slot modes (no writes through replicated slots; writes only into the body's own split view).
15. R15 FORM-1 (Fail-closed form table). Every operation of a loanable sealed form carries exactly one loan clause; a call whose loan behavior is neither declared in the form table nor derived by R9/R10 is rejected — absence of declaration is rejection, never default-accept. An op may be declared with & receiver mode on a loanable form only if its internals both preserve every interior address and are safe under concurrent invocation; these obligations, the release actions run by holder drops, and issues-on-source(receiver) clauses are sealed-form proof obligations subject to hostile fact-channel review before shipping — a green checker is not that review.

## Machine spec

CHECKER STATE (per function). (1) Bindings: map name -> {type; mode in {owned, reference(region)}; status in {live, consumed}; decl_index; if confined: kind(type) in {uniq, shr} read off the declaration, and src: map from each region parameter of the type to a resolved place}. (2) LoanTable: set of entries (place P, kind k, holder h). A resolved place is (root binding, field path — possibly empty); overlap(P,Q) iff same root and one path is a prefix of the other. (3) Block stack with per-block declared-binding lists. (4) During matches/loops: saved snapshots (deep copies of 1+2) and collected break/return states. Nothing else is mutable state.\n\nDECLARATIONS PRE-PASS (runs before any body; all failures reject the declaration). (a) Confined types: only confined types may contain borrow or confined fields; every region used by a field appears in the parameter list; loanable confined types have >=1 region parameter; kind coherence — a confined(shr) type must not transitively contain a &uniq field or a confined(uniq) field. Non-confined types containing either are ill-formed; after monomorphization, re-check every instantiated type (catches generic wrappers and sealed-form type arguments — instantiating any form/generic type parameter with a confined type rejects). (b) Signature ties: for each region parameter 'p of each confined parameter or confined result: let D = distinct borrow-mode parameters (not the confined parameter itself) whose declared region is 'p. If |D| = 1: tie('p) = that parameter. If |D| = 0 and the confined parameter is itself passed by borrow: tie('p) = SELF. Otherwise reject. Record the derived effect: consumes = the own-mode confined parameters; issues = for each confined result region 'p, (tie('p), kind(result type)). (c) Form table: every op of a loanable form must carry exactly one clause — issues K, issues K on source(receiver), consumes, reissues, none; any call not covered by a clause or a derived signature rejects (fail-closed). & receiver ops on loanable forms carry the address-stability + concurrent-safety obligation (review gate, outside the checker).\n\nSTATEMENT TRANSITIONS (one pass, ANF order; each statement: resolve operand places -> legality -> consumes -> issues -> liveness updates).\n\n[Mint legality, used everywhere] Minting & of place Q: reject if any uniq entry overlaps Q, unless EXC. Minting &uniq of Q, or moving/own-passing/assigning/explicitly-dropping Q: reject if ANY entry overlaps Q, unless EXC. EXC (consume-exception): this statement consumes or reissues a holder h, h's entries are the ONLY entries overlapping Q, and the minted borrow of Q is declared by the callee's signature. A live confined binding mints at most one statement-scoped borrow per statement; field projection through a live confined binding or a borrow of one yields only a statement-scoped borrow consumed by the receiving call; both transfer no entries.\n\n[let x = f(args) / op call] (i) For each confined argument h (own or borrowed), for each region 'p of its type with tie('p) != SELF: let R = the resolved place of the argument bound to tie('p); require status(h)=live, src(h)['p] = R, and (R, kind(h), h) in LoanTable; else reject. SELF ties: skip the comparison; additionally, passing a self-tied-origin confined value into a distinct-tie slot rejects (its src is unknown — reject-when-unsure). (ii) Mint legality for every borrow argument per above. (iii) Consumes: for each own confined argument consumed by the clause/derived effect: remove all entries with that holder; mark it consumed. Other own arguments: require no overlapping entries; mark consumed. (iv) Issues: if the result type is confined, for each region 'p of it: source place S = place of the argument at tie('p); for clause `issues on source(receiver)`, S = src(receiver-holder)[the type's region]. Precondition (checked AFTER consumes): kind uniq -> no entry overlaps S; kind shr -> no uniq entry overlaps S; else reject. Add (S, kind(result type), x); set src(x)['p] = S. The result must bind a fresh let or a rebind. Region for escape accounting: region(S) = declared region of S's root if it is a reference binding, else the block region of the root's declaration.\n\n[let x = ConfinedType { fields }] Every region parameter of the type must be instantiated by a captured borrow mint or a moved confined field in this construction; any unbound region parameter rejects (no captureless/forged instantiation). Moved confined field h: rename all entries holder h -> x; merge src(h) into src(x); mark h consumed. Captured borrow of place S with mint mode m: mint legality; precondition and entry addition as in (iv) with kind = kind(ConfinedType); src(x)[the field's region] = S. Kind never changes in any transfer.\n\n[let x = move h] and [rebind x = h] (x previously consumed, identical confined type): rename entries holder h -> x; copy src; h consumed, x live. Assigning over a live confined binding rejects.\n\n[match scrutinee { arms }] If the scrutinee is a live confined binding: require it is an enum whose EVERY variant packs all borrow/confined payload into exactly one confined field of the same kind and region parameters as the scrutinee's type; else reject (struct patterns on confined values always reject). Consume the scrutinee; in each arm, rename its entries to the arm's bound token and copy src. All matches: snapshot state; check each arm from a copy; at arm end run block scope-end processing for arm-locals (below); arms ending in break/return contribute their post-processing state to that exit's join instead. Compare fall-through arm states projected to bindings declared outside the match: identical LoanTable entries (including holder names) and identical liveness, byte-equal, else reject.\n\n[loop { body }] Snapshot entry state; check body once; after body-local scope-end processing, if the back edge is syntactically reachable (not every path ends in break/return — a syntactic test over match-only ANF), require the projected body-end state to equal the entry state exactly, else reject. All break states must be pairwise equal (same projection) and become the post-loop state.\n\n[return e] If e is confined: every region parameter of its type must be a region parameter of the enclosing function, else reject. Run scope-end processing for all locals except e (reverse order). Then exit-effect check: the remaining LoanTable restricted to parameter roots must equal exactly the signature-derived caller effect — entries of consumed own-confined parameters gone, and for each result region 'p exactly (tie('p)-binding, kind, e); any residue or deficit rejects.\n\n[scope end of a block or function] Process the block's bindings in REVERSE declaration order: a live confined binding is auto-consumed — emit its form-declared release (unlock, close, retire) and remove its entries; then, for each owned binding being dropped, if any live entry overlaps its place, REJECT (derived drops obey the freeze); otherwise drop per STOR-3. Reference parameters expire without drops; entries seeded at function entry expire silently at function exit.\n\n[function entry seeding] For each confined parameter h and each region 'p with tie('p) = B (distinct): add (B, kind(h), h) to LoanTable and set src(h)['p] = B. Self-ties seed nothing. Own-mode confined parameters not consumed by the body are auto-consumed at scope end (release runs in the callee — consistent with the caller's derived consume).\n\n[par calls] Per the form's slot marks: replicate-shared slot — argument must be a & mint of a place with no overlapping uniq entry, or a & mint of a live holder all of whose held entries are shr (a holder with any uniq entry rejects); split-unique slot — one &uniq mint, standard mint legality. Body function's effect rows must match slot modes. All mints are consumed by the statement; no entry changes unless an argument is consumed by clause.
