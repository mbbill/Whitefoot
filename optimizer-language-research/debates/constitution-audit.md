## Tensions with the constitution

- Structural D2-R3 tension: the round-4 decided law makes the spec-delta column mandatory for every proposal (META-5), so the spec-cheapest candidate wins gate review by default -- exactly the selection basis R3 forbids ('not the cheapest to specify'). The gate discipline lacks the tie-break rule that spec-delta PRICES candidates but AI-codegen + performance evidence SELECTS them. Root cause of most provisional items above.
- R2/R3 vs gate discipline -- a live breach: type-inference and interfaces both sit at needs_evidence in the round-2 table (verified this merge) and were never promoted, yet TYPE-5 bans interior inference (via D2 spec-cost reasoning) and FN-3 ships the interfaces replacement as normative. Decided without the evidence the gates required; the constitution makes this the clearest procedural violation on record.
- R2 vs the metaprogramming ban while its replacement is deferred: round-2 'metaprogramming: replace_with_alternative' is law, but the alternative (closed constant-expression sublanguage) is DEFERRED -- CONST-1 admits integer literals only. R2's own cited natural experiment (Go pre-1.18: go:generate as unchecked external templating) is precisely the failure mode invited for as long as the gap persists.
- R1-performance tension at the loop form: GRAM-6 iteration surfaces zero optimizer-visible loop facts, while the round-1 arrays-loops leading candidate (gate: prototype_needed) established that declared loop kinds supply the A005 consumer list by construction. The kernel froze the weakest-fact iteration form before the discriminating prototype ran -- the performance half of R1 is currently unserved by the retained form.
- R3 vs ratification sequencing, with breaking-change urgency: kernel v0.2 hardened forms whose deciding experiments round 2 pre-registered but never ran (syntax A/B, redundancy-independence, drift experiments), and the current ratification preconditions (sect. 5 formal-calculus reconciliation, sect. 9 effect carding, owner counting rulings) exclude these R3 debts. Because FORM-1 admits exactly one spelling per construct, a later evidence-driven switch (e.g., GRAM-7 to expression-match; EX-1 bakes the helper idiom) is a BREAKING canonical-form change -- the R3 audit is time-urgent, not backlog.
- R1 (feedback-loop speed) vs PROG-1 closed world with no separate compilation plus FN-2 re-checking every monomorphized instantiation as concrete code: check-loop latency is an enumerated R1 value, but no gate entry measures or budgets whole-program recompile cost at target scale.
- R1 (feedback-loop speed) vs FORM-1's toolchain-never-auto-formats hard error: every byte-level formatting deviation costs a full generate-reject-regenerate round trip; an accept-and-canonicalize alternative was never evaluated.
- R4 (trapping is the fallback) vs round-3 'wrap fallback legal' wording for overflow: silent wraparound is a silent-logic-error channel. Currently defused because OP-1 has no default (writer names .wrap/.trap/.checked per node; OP-2 excludes wrap where unsound), but any future defaulting rule instantiating 'wrap fallback' would violate R4 and META-2 simultaneously; the round-3 wording should be annotated now.
- R6 vs the undecided ir-strategy verdict: round-1 'ir-strategy: research_needed' is still open while the normative operation table (OP-1) and artifact rules (DIAG-2 'acceptance decidable from the artifact alone') accrete; only the lexicon (LEX-1) has been audited for backend coupling -- the v0.2 Rust-divergence audit does not cover operation- or artifact-level coupling.
- D2 budget vs R4-load-bearing diagnostics: the no-shadow-spec rule taxes precisely what R4 prioritizes -- DIAG-3 report field schemas and ERR-3 verifier-checked Result propagation + Result-vs-trap classification were deferred as counted spec mass, yet under R4 + R1 the check/lifetime/trap report family and the classification rule are load-bearing for keeping recoverable errors shift-left, not deferrable luxuries.
- D1a-R2 tension at the checker levers: OWN-3 lexical regions and OWN-8 reject-when-unsure are grounded in the D1a effort ceiling, but under R2 a checker cut that inflates rejections of sound programs harms AI codegen; rejection-rate / restructuring-churn measurement should accompany the sect. 5 formal-calculus reconciliation its PROVISIONAL marker already requires.
- R1-unaccounted human-residue rules (audit flags): (a) FORM-2 indentation/blank-line/spacing conventions encode information already carried by braces and separators -- training-distribution-alignment defense exists but is uncarded; (b) FORM-4/GRAM-3 doc STRING field -- human-readable prose in a language whose text humans are declared not to read (R5), its only defensible grounding (writer re-read context, R5 gate-audit support at LEDGER-1 signatures) nowhere recorded; (c) free-text message in GRAM-4 check_stmt / SCOPE-4 trap reports, where DIAG-3 already mandates rule ID + node path. None load-bearing for soundness; silent human-serving rules are the one thing R1+R5 jointly forbid. Related R2 near-miss note: GRAM-7, no-if, and TYPE-6 are the most plausible R2-reversal sites if experiments land badly -- the same cut-for-simplicity move as the generics case, made WITHOUT a checker-collapse-grade argument.
