# Round 3 — Safety-envelope synthesis

One coherent envelope under D1 (races/UAF impossible) + D0 + round-2 floors. Composition verdict: the four rulings compose given ONE spine amendment, three ordering constraints, and two shared blockers.

## Envelope architecture (Layers 0-4 + invariants)

- LAYER 0 — PROVEN (assumed in every build mode, zero runtime cost). Contents: everything the deterministic checker derives from the decided floors — checked ownership/borrowing and storage/lifetime contracts (F001, F006, C004 now CONFIRMED_WITH_AMENDED_SOURCES 2026-07-02), Send/Sync-class capability predicates, mono-only generics checked once against declared bounds, exhaustive-match dispatch, closed-world whole-program facts (D004/D007/D008), and compiler-derived facts about constant data (metaprogramming law). Writer-stated intent facts at boundaries (ownership/aliasing capability, effect row, numeric mode, ABI-visible layout) are inputs re-verified against bodies every build — nothing writer-stated is ever trusted unchecked (verification spine). Two enlargement channels feed this layer: (a) an OPTIONAL per-fact-class stated-and-checked channel (loop invariants, ranges, loop-bounded disjointness, law obligations) — fact-creating for checker-underivable facts but always checked, never trusted (proof-burden ruling amendment 1); (b) a deterministically-budgeted solver PROMOTION stage that may enlarge the proven set on the performance ledger only — it never gates acceptance and never elides safety-relevant checks until a proof-producing-solver + small-independent-checker TCB design exists (proof-burden amendment 2).
- LAYER 1 — CHECKED (runtime checks in ALL build modes; tested artifact = shipped artifact). Every unproven D1-critical checkable fact — array bounds, and alias-disjointness where a check form exists (A005 loop-bounded pointer-range guards only) — carries a runtime check lowering to the decided trap-with-report=abort; elimination only by Layer-0 proof. All checks, writer-spelled or compiler-derived, are surfaced as explicit ops in the canonical elaborated artifact plus machine-readable per-function check reports, on the round-2 MEMORY drop-op template; check-density/elision reports are the lint-class containment channel. Integer overflow is a POLICY class inside this layer, not a D1 class: named truthful modes are law, unproven nsw emission is spine-forbidden, unflagged wrap is a legal sound fallback, and the bare-'+' default binding stays with the open numeric-semantics question. Semantics identical across build modes is the target property, currently undefined at two points: relaxed-trap-ordering (vectorization legality of checked inner loops) and trap semantics under racing threads — both must be specified before this layer's spec closes.
- LAYER 2 — GATED (the single boundary-construct family; the only home of trusted assertions). Exactly ONE family in core surface: unsafe regions, FFI extern declaration frames, trusted primitive/capability-base imports (mutex/atomic/allocator base impls per C004), and reflection capability grants (round2-modules line 59) — sharing one per-fact soundness-obligation ledger (the round1-ir-strategy artifact). A bare or manifest-free member is unrepresentable (P2's form); every entry carries content-addressed identity, provenance, rationale, and named obligations. The steady-state AI writer can neither author nor edit any member: authorship is a privileged, gated toolchain operation extending the decided contract-edit-gating floor, so NO writer-emittable believe-me construct exists anywhere in the language (P1's invariant preserved inside P3's skeleton). Discharge preference is mechanical: static proof > A005-style runtime check under trap=abort > machine-checked proof artifact; review-only authorization is confined to obligations with no mechanical discharge, quota-logged and density-measured per round2-memory-automation. One family + one ledger closes performance-pressure arbitrage between unsafe, FFI, and primitive channels.
- LAYER 3 — QUARANTINED (the conservative wall; imprecision costs optimization, never soundness). Every extern crossing carries a mandatory machine-readable declaration frame (reads/writes/escapes/threads/no-unwind/re-entry — synthesis line 23, round2-modules line 25), but the only frame admissible to writer-reachable code is the maximally conservative one; any stronger frame exists only as a Layer-2 gated-family member with discharged ledger obligations. C003-style barrier semantics at every crossing; any memory foreign code may retain must be PRE-DECLARED foreign-shared at its allocation site as a field on the existing storage-contract floor (compiler-inferred call-site demotion is rejected as a floor violation); mandatory LangHeap pointer-integer laundering ban / provenance rule as the forced companion decision (without it, barrier imprecision becomes a soundness hole — threat-model row 2); foreign unwinds abort at the boundary. The closed world bounds the invalidation set: F001/F002-class facts hold at full strength over the FFI-unreachable interior.
- LAYER 4 — AXIOMATIC (outside the envelope, honestly declared, not silently assumed). The ambient trust that no in-language mechanism discharges: ABI-well-behaved foreign code (a misbehaved foreigner can scribble on the address space under every design), and the delimited-but-currently-UNdelimited TCB — allocator, scheduler/runtime, OS syscalls beneath LangHeap, plus the compiler/verifier/checker implementation itself. The honest position-invariant envelope statement, recorded now: memory- and thread-safety bugs are impossible in accepted programs CONDITIONAL on ABI-well-behaved foreign code and the declared TCB — exactly Rust's shipped envelope (F001 reference bar). Only hardware/OS isolation (process, MPK, CHERI — zero cards) strengthens the conditional itself; P2-FFI's 'isolated' bottom-tier contract stays expressible so the escalation path exists.
- CROSS-CUTTING INVARIANTS binding all layers: (1) writer-trust invariant — the steady-state AI writer never authors trust at any layer; every writer statement is checked, quarantined-conservative, or unrepresentable; (2) artifact honesty — every safety-relevant operation and fact source (checks, drops, frames, ledger entries, discharge records) is visible in the byte-canonical elaborated artifact and its machine-readable report family (check reports, lifetime reports, discharge ledger, wall-crossing logs feeding trap reports); (3) one gating regime — contract edits, boundary-family authorship, assume-mark addition, and FFI frame changes are all the same privileged toolchain operation; (4) acceptance is artifact-checkable, not process-relative — proof objects and check instrumentation travel with the program so D1's 'accepted programs' quantifier stays stable across toolchain versions.

## Decided now

- D1 + spine + trap=abort force proven-else-checked for D1-critical checkable facts: an unproven bounds or alias-guard fact must be checked in ALL build modes, proven, or the program rejected — ambient assumption is excluded; failure semantics is trap-with-report=abort. This floor is benchmark-robust: no cost measurement can reopen ambient assume, only route pressure into proof, restructure, or the gated family.
- No free-form writer-emittable unsafe block exists in any admissible design — forced jointly by the verification spine (no third state), the contract-edit-gating floor a fortiori, N001's unbounded blast radius, and D0's least-trusted-writer premise; no advocate in any of the four debates actually defended one.
- A hatch EXISTS, as exactly one gated fact-boundary construct family (unsafe regions + FFI extern frames + primitive/capability imports + reflection grants) with one per-fact obligation ledger; manifest-free members unrepresentable; authorship/editing is a gated toolchain operation the steady-state writer cannot perform. Existence is forced: round1-concurrency's 'trusted-base placement, not elimination', C004's base impls, and the threat model's unverifiable rows mean something hatch-shaped is authored somewhere; in-language placement preserves partial verifiability (threat-model 'sometimes no') that FFI relocation forfeits ('no').
- All checks are artifact-surfaced on the MEMORY drop-op template: compiler-derived-but-artifact-honest operations are decided law's own pattern, so 'checks visible in the elaborated artifact' is entailed by canonical form — and 'no compiler-inserted check' (check-policy P3) is not. Machine-readable per-function check reports join the report family now.
- Conservative no-emit fallback is law where no check form exists (general whole-execution noalias, purity) and at FFI/int-ptr boundaries — round-1 trilateral consensus, threat-model remedy column, reaffirmed by all four rulings.
- Overflow is classified now as a non-D1-critical policy class: unflagged wrap is a sound, spine-compliant fallback (N002); unproven nsw emission is forbidden; any unchecked/assume mode must live in the gated family (instantiating the numeric-semantics backlog note 'unsafe/proof-gated'). Only the bare-'+' default binding remains open.
- Mandatory FFI declaration frames at every extern crossing, with only the maximally conservative frame admissible outside the gated family; C003 barrier semantics; foreign unwinds abort at the boundary (hardening the previously provisional round-2 cut-list line).
- Foreign-retainable memory is pre-declared foreign-shared at its allocation site — one field on the existing writer-stated storage-contract floor; compiler-inferred call-site demotion is REJECTED (violates explicit-storage-contract, zero-cross-boundary-inference, and contract-edit gating). The LangHeap pointer-integer laundering ban / provenance rule is adopted as a forced companion decision: every wall design is unsound without it.
- Mandatory writer proof burden = exactly the decided round-2 floors and nothing more; nothing writer-stated is ever load-bearing unchecked. The stated-and-checked channel above the floor is OPTIONAL and per-fact-class; P1-style mandatory proof-carrying source is ineligible as a mandate (violates the compiler-derived-constant-facts law as written, non-canonicalizable proof surface, and D2's amended verbosity budget).
- Solver placement is fenced now: a deterministically-budgeted (rlimit-class, pinned-version) solver may only PROMOTE facts on the performance ledger; it never gates acceptance and never elides safety-relevant checks until a proof-checker TCB design exists. 'Elimination only by proof' in the check layer means deterministic-checker-grade or verified-proof-artifact discharge, not raw SMT output.
- The honest conditional envelope statement (Layer 4) is written into the safety-envelope record NOW — it is position-invariant across every design considered in all four debates, so nothing is lost by recording it before the pending D1-quantifier owner ruling.
- One formal spine amendment is adopted as the single reconciling legislative act (see interactions): 'no third state' is restated as 'no WRITER-EMITTABLE third state; toolchain-gated ledger entries are the sole trusted-assertion class, named, quarantined, discharge-preferred-mechanical, quota-logged, density-measured'. This simultaneously resolves synthesis line 23 vs line 33, check-policy's T2 remand, and the hatch ruling's review-only residue — three rulings, one amendment.
- Until that amendment is formally issued and its preconditions carded, the operative default is the strictest reading: proven-else-checked with no trap-free assume anywhere, conservative-only FFI frames, and a near-empty gated set (bootstrap primitive base only).

## Residual unsoundness surfaces (honest enumeration)

- ABI-misbehaved foreign code: a foreigner that violates the call-boundary axiom can scribble on the entire address space under EVERY design — blast radius unbounded in principle, identical to Rust's shipped envelope (F001 bar). Only uncarded hardware/OS isolation (process/MPK/CHERI) removes it. Size: unquantifiable until the interop workload census exists; position-invariant, so it prices no design choice.
- Undelimited TCB: allocator, scheduler/runtime, OS syscalls beneath LangHeap, plus compiler/verifier/checker implementations, are trusted with no written declaration. Size unknown and currently unbounded-by-omission; the hatch ruling's frontier-sizing decides whether the primitive base is 'a standard library' or 'a second evidence-free language'. No card exists on checker/TCB defect history — every TCB-growth argument in the corpus is structural reasoning.
- Review-only gated residue: ledger obligations with no mechanical discharge (whole-execution noalias, purity, Send/Sync base-impl claims, non-conservative FFI frames) are relocated trust — review, human or AI, is a trusted assertion (P3 critic, confirmed). Size = frontier size, THE decisive unmeasured variable named by round1-aliasing, memory-automation's reopening clause, and the backlog; same-model review shares correlated errors (unresolved).
- Obligation-generation completeness: a missed obligation class is a hole no checker, prover, or gate closes — the SHARED soundness surface under all four rulings. Zero cards, no validation plan. Size: unknown; it dominates every layer's soundness story and is the architecture's single largest unexamined surface.
- Caller-drift falsification of boundary assertions: whole-program regeneration rewrites safe code around hash-stable Layer-2 regions whose context-dependent assertions (noalias, non-escape, Send) can be falsified by new callers while every provenance and hash check passes. Re-validation triggers are undesigned for ALL forms. Size: scales with regeneration frequency — plausibly the highest-probability real-world breach channel for an AI-regenerated codebase, and completely unmeasured.
- Inbound FFI direction: foreign-supplied values (uninitialized buffers, invalid representations, aliasing-contradictory out-params) have no threat-model row and no design; language-side marshalling from a never-written foreign buffer is literally a D1-banned uninitialized read. Size: every inbound crossing until designed — a hole in the wall, not a conditional axiom.
- Foreign threads executing language callbacks: touch LangHeap by construction; no card defines capability predicates for runtime-foreign threads, callback-entry identity checks, or exported-callback env-struct residency under the closures floor. This is the closed-world reopen trigger arriving through the wall. Size: every callback-driven interop workload.
- Trap semantics under racing threads: unspecified (round-2 debt, answered by no position) — abort ordering, report atomicity, traps on FFI-introduced threads. Caps the failure/observability story of Layer 1 at the boundary. Size: bounded (it is a spec hole, not a mechanism hole) but blocks the check tier's spec from closing.
- Relaxed-trap-ordering: absent a sound spec validated against LLVM's speculation/side-effect model, either checked inner loops face a vectorization cliff (strict ordering — a D0 cost, not unsoundness) or a naive relaxation is itself a soundness bug. Size: bounded, prototype-gate class; but Layer 1's headline property is undefined until it lands.
- Provenance/int-ptr rule: adopted as a decision but not yet designed (memory-object-model open questions). Until specified, reachable-set completeness fails and Layer-3 imprecision is a silent soundness hole rather than an optimization loss (threat-model row 2 'often no'). Size: the entire wall until ruled.
- Post-violation attribution hole: an ABI-axiom violation manifests as a mis-attributed trap in innocent internal code — an observability defect, not a new unsoundness, but it silently corrupts the audit/blame channel every gated-residue argument leans on. Wall-crossing logging is undesigned.
- Interior-mutability capability type: every position's mutex/atomic/channel signatures need an UnsafeCell-analog whose tension with shared-reference-noalias is uncarded (misleading-comparisons #2); round2-memory-automation bars admitting it before carding. Until then the type-level half of the escape envelope is an IOU sitting inside the primitive base.
- Verifier/solver trust-path creep: the solver fence is asserted design, not implemented mechanism; if solver-discharged facts ever elide safety checks before the proof-checker TCB exists, a solver bug becomes a D1-class breach. Size: zero if the fence holds; the fence itself is unenforced until built.
- Honest global sizing statement: no residual surface above is quantified anywhere in the corpus (caveats.md); the three that dominate expected risk — frontier size, obligation completeness, caller-drift revalidation — are respectively unmeasured, unexamined, and undesigned. All AI-writer behavioral claims touching these surfaces remain EMPIRICAL-UNTESTED (round-2 standing flag, zero AI-codegen evidence).
