# Round 4 (v2, corrected D2) — Compiler as teacher (`compiler-as-teacher`)

**Outcome:** `research_needed`

**Leading candidate:** Spec-primary-leaning hybrid on P3's delivery mechanism with P1's diagnostic guarantees as law: a D2-capped kernel spec (few uniform rules + canonical worked examples + the diagnostics/report vocabulary, which must be taught eagerly) generated and versioned as a provenance-checked teaching pack; the repair loop carries only prior-correction, mechanical edit-list propagation, and the conceded nonlocal-ownership residue; every diagnostic is deterministic, one-pass-complete, aggregated, and rule-cited pointing back INTO the spec (reinforcement, not substitute curriculum); checker-invisible quality classes route to by-construction design + differential/silent-wrong harness + lint channels, never to either teaching channel; the exact per-rule split is set by the pre-committed ablation experiment, and P3's writability benchmark runs advisory (not shipping-gate) until the transfer-validity study passes.

## Rationale

The question is genuinely empirical and currently unanswerable to the project's own standard: the corpus has zero AI-codegen cards, the two most-cited teaching anchors (D009, C002) are unverified provisional, and the critic exposed that the debate's core quantity lacks a unit — D2's text (user-directives line 16) binds the language description as an artifact, which already caps the rule count regardless of allocation, but no owner ruling says whether diagnostic-delivered elaborations count against that budget. That definitional gap plus the pending pedagogy experiments force research_needed. Post-critique, however, the positions converge on more than they contest. First, a shared toolchain-guarantee floor survives all attacks and should be adopted now: stable content-addressed rule IDs; deterministic byte-stable diagnostics carrying rule ID, canonical rule text, node-path witness, and edit list where mechanical; complete one-pass error reporting; one-defect-one-repair-site; a rule-to-emitter coverage audit; loop instrumentation; and a single unified telemetry + gated spec-evolution channel (three were proposed; shipped separately they violate the near-duplicate-machinery charter). Second, two structural facts bound the split before any experiment runs: (a) the checker cannot teach what it cannot reject — the D0-critical silent-wrong classes and performance idioms need eager spec/examples plus a differential harness, so the loop can never be primary for what actually matters; (b) the adopted soundness-over-completeness lever creates restructure-demand lessons that are neither spec citations nor mechanical fixes, breaking P1's bijection exactly on the hardest (ownership) errors, while the decided Miri-style dynamic oracle is an unplaced fourth channel. Third, under a D2-compliant few-rules language the eager-vs-lazy token delta shrinks, and the corpus's nearest evidence (K001 Austral spec philosophy, F006 SPARK) favors small-kernel eager feasibility — so the burden of proof sits on moving teaching INTO the loop, and the loop's defensible steady-state role is prior-correction, mechanical propagation, telemetry, and a bounded named ownership residue. P2's direction therefore leads, but stripped of its manufactured legs (the floor-vs-ceiling misreading, the zero-marginal-cost inversion, the three-way-confounded irregularity meter) and mounted on P3's generated-pack mechanism, which is the decided architecture's honest extension for code examples (drift-free) though not for prose. P3's CI gate stays advisory until the benchmark-transfer study passes, because gating language evolution on current-model pass rates is exactly the process-relative acceptance round-3 invariant 4 resists. The named experiments, with the D2 unit ruling first, convert the remaining disagreement into a measured per-rule table.

## Position assessments (post-critique)

### Repair-loop-primary (kernel spec + checker-as-teacher via citation-grade diagnostics)

**Strengths:** Its toolchain-guarantee set largely survives independent of its allocation claim: stable content-addressed rule IDs, diagnostics carrying canonical rule text + node-path witness + edit list, determinism/byte-stability, one-defect-one-repair-site, rule-to-emitter coverage audit, and loop instrumentation are all cheap extensions of machinery round 2 already mandates for correctness (round2-synthesis lines 66-76), and the decided floors (canonical-form hard errors, zero implicit conversion/inference, banned wildcards, no writer-emittable trusted assertions) genuinely make acceptance track spec-conformance for checker-visible rules — the precondition ordinary languages lack (F006 apt; D009 apt as contrast, though unverified). Honest concessions; only position that designed the migration dial in detail.

**Weaknesses:** Three critic hits land hard. (1) D2 binds the language DESCRIPTION as an artifact (user-directives line 16 verified): if a 'long tail of rule elaborations, interaction cases, edge semantics' exists to warehouse in a citation database, the language already violates D2 wherever the text lives; under a D2-compliant few-rules language the eager-vs-lazy delta shrinks and the allocation argument loses magnitude — worse, lazy delivery removes the cost pressure that enforces the rule cap. (2) The rule-to-diagnostic bijection is broken by decided architecture it never cites: the adopted soundness-over-completeness lever (decision-gates line 74, verified) guarantees rejections that cite no violated rule and admit no mechanical fix (restructure demands), concentrated exactly on the nonlocal ownership errors everyone concedes are hardest; its diagnostic taxonomy cannot represent this class. (3) F003 is cited inverted (the card marks the frontend fact boundary as the soundness HAZARD, not a trust warrant), and 'same defect → same bytes → repair converges' states an untested model-behavior claim as fact while its own risk 3 concedes livelock is undesigned-against. The dynamic half rides unspecified concurrent-trap semantics (round3 lines 53-54), and diagnostic text becomes an unpriced N006-grade teaching TCB.

### Spec-primary (compact uniform spec carries the burden; loop = safety net, mechanical executor, telemetry)

**Strengths:** Owns the argument that survives even if loops become free: the silent-wrong asymmetry. The checker can only teach what it rejects, and the record's own verdicts enumerate checker-invisible classes — region pessimization/leak holes (round2-synthesis lines 60, 74), context-dropping Result variants (round2-error-handling), operand-order swaps (round2-syntax), and performance-wrong code generally, which is D0's actual objective — so a nonzero irreducible eager burden exists regardless of loop economics. Extending round-3 invariant 4 (artifact-checkable, not process-relative) to pedagogy is a genuine architectural harmonization: spec competence is version-pinned; repair-session competence evaporates at context boundaries. D2's few-rules cap plus uniformity-as-transfer, F006/SPARK, and the uncited K001 Austral card (learn-the-whole-language-from-the-spec philosophy) make the small fully-taught kernel the best-precedented feasibility story in the corpus.

**Weaknesses:** Two legs are manufactured or inverted, as the critic showed. Leg 2 litigates silence as law: round 2 decided a diagnostics FLOOR, not an explanatory-content ceiling, and P2's own obligation (b) (rule-cited diagnostics) proves it believes the channel extensible. 'One spec line amortizes to zero marginal cost' prices the debate's contested quantity at zero in its own favor — every task re-pays the in-context payload, which is exactly the cost structure D2's motivation clause names. The irregularity-meter (leg 3) cannot discriminate its three causes — D2-irregularity vs prior-interference against a uniform rule vs D1a checker incompleteness rejecting sound patterns — yet routes all persistent error mass to language redesign; that is an uncarded distributional claim at the position's operational core. And the silent-wrong asymmetry, its best leg, actually routes burden to a THIRD channel (by-construction language design + differential/silent-wrong harness, per the record's own decided remedies), not to spec prose; 'can only live in the spec' overreaches, though 'cannot live in the loop' stands.

### Curriculum-as-build-artifact (generated, budgeted, CI-gated teaching pack; split discovered by ablation)

**Strengths:** The mechanism is the decided architecture applied to teaching: a deterministic provenance-checked generator (round-2 consistency mechanism 2) producing content-addressed views over the canonical substrate, with example extraction made sound by the single-spelling floor — extracted examples are re-verified subtrees that cannot drift. Its per-rule ablation experiment (drop rule text, add repair example, measure deltas under fixed budget) is the most precise answer-procedure any position offers for 'how much burden moves', and converting the split from doctrine to a measured knob matches the project's own epistemics (pre-committed thresholds, gate-before-promote). Only position that prices the missing consideration that the diagnostics/report vocabulary is itself spec mass a writer must be taught eagerly.

**Weaknesses:** Systematically overclaims, as the critic documented. 'Only position with a measurement mechanism' is false (P1 ships loop instrumentation, P2 a fleet ledger and spec-ROI curve); its true distinctive — CI-gating language changes on current-model pass rates — is precisely the process-relative acceptance round-3 invariant 4 resists, plus model-idiom lock-in (baking 2026 priors into permanent design). 'The pack is D2's regression test' is a category error: pack tokens and pass rates measure size and model difficulty, not irregularity, D2's stated metric; a uniform verbose rule grows the pack while being owner-ruled spec-cheap. The C004 gloss contradicts Layer 1 (unproven D1-critical facts are runtime-checked under trap=abort — dynamic, one-lesson-per-process-death, not 'complete compile-time lessons'). 'Cannot drift' covers only code subtrees — kernel prose and anti-prior notes can rot, and pack-faithfulness validation is a conceded undesigned deliverable. The closed diagnostics vocabulary cannot close over an open spec (concurrent-trap, bare-'+', interior regime), making 'novel free-text errors are toolchain bugs' vacuous or a permanent blocker.

## Key tradeoffs

- Eager vs lazy delivery economics: an eager spec pays a fixed per-task context cost for rules most programs never touch; lazy diagnostics pay per-violation full generate/check/re-read iterations (and, for Layer-1 dynamic lessons, one process death each under trap=abort). Unmeasured in both directions; the heavy-tail violation-distribution premise decides the magnitude, and D2's few-rules artifact cap shrinks the delta either way.
- Coverage vs economics: the repair loop can only teach checker-visible rules; the classes that matter most under D0 (performance-wrong code, region pessimization, context-dropping, operand swaps) are rejection-invisible by construction, and the adopted soundness-over-completeness lever adds sound-but-rejected restructure demands that are neither spec-rule citations nor mechanical fixes. So allocation can never be purely loop-side regardless of who wins the token economics.
- Diagnostic richness vs diagnostic trust: teaching-grade diagnostics (rule text + witness + fix) accelerate repair if the ablation confirms it, but convert diagnostic text into a TCB-adjacent surface where a wrong citation actively teaches a false rule — demanding N006-grade tested-and-versioned maintenance, an unpriced cost carried in proportion to how much teaching the loop does.
- Artifact-checkable vs process-relative competence: spec-encoded knowledge is version-pinned, auditable, and identical fleet-wide (extends round-3 invariant 4); repair-session knowledge is per-instance, washes out at context boundaries, and is re-paid per session unless a distillation channel works — and every distillation channel is itself a D2 accretion vector needing the unified gating regime.
- Writability gating vs model lock-in: CI-gating language changes on current-model pass rates catches teachability regressions before shipping (J002-style guard) but re-instantiates process-relative acceptance and risks baking today's model priors into permanent language design.
- Toolchain proliferation vs charter: the three positions propose three overlapping telemetry systems and three privileged spec-evolution channels over the same checker; shipped separately they reproduce the near-duplicate-machinery drift the charter forbids in source — unification is required, not optional.
- Loop-latency dependence: any loop-side teaching share puts the closed-world whole-program check (round-2's biggest unpriced cost) and the multi-process trap=abort harness on the critical path of every lesson; if the content-addressed-caching and wall-clock prototypes miss thresholds, the allocation inverts toward the spec regardless of pedagogy results.

## Evidence gaps

- Zero AI-codegen evidence cards exist (standing round-2 flag, caveats.md): every claim about how models learn a novel language from in-context spec + examples — the debate's entire subject — is uncarded in all three positions.
- The two cards doing the most teaching-argument work are unverified provisional: D009 (silent degradation) and C002 (post-violation observability) are in the pending adversarial sweep (decision-gates.md line 49; round3-synthesis line 78). Only C004's status was correctly carried, by P3.
- K001 (Austral, direct_fetch_extracted 2026-07-02) — the corpus's only card touching learn-the-whole-language-from-the-spec philosophy and the adopted minimal-calculus requirement — was cited by no position despite being the nearest actual evidence on kernel sizing and spec-primary feasibility. It is spec-level only; no implementation- or learning-outcome evidence.
- D2's binding unit is undefined at the decisive margin: user-directives line 16 binds the language DESCRIPTION (artifact), which caps the rule count regardless of allocation, but whether diagnostic-delivered elaborations, witnesses, and anti-prior notes count against that budget — versus being toolchain output — has no owner ruling. The shared arbitration experiment cannot be scored until 'spec' is defined against 'citation payload'.
- Rule-violation frequency distribution (the heavy-tail premise of lazy teaching) is unmeasured — no census exists.
- Whether models act on lint-class non-error feedback (round2-synthesis line 104) is untested and gates the only proposed teacher for the checker-invisible quality classes.
- Closed-world whole-program check-loop latency at AI-scale program sizes (round2-synthesis line 98) and multi-process trap=abort harness wall-clock (line 72; round2-error-handling line 26) are both unpriced — joint viability gates for any loop-side teaching share.
- The restructure-demand lesson class created by the adopted soundness-over-completeness checker lever has no diagnostic design, no spec chapter, and no pack vocabulary in any position; the round-3 battery names the experiment but no teaching-side owner exists.
- The Miri-style dynamic-oracle cross-check (decision-gates line 74) is a decided fourth teaching channel — concrete dynamic witnesses for ownership/lifetime errors — placed in no position's allocation.
- Diagnostic-content effect (rule text + witness + fix vs rule-ID vs bare message) on repair speed, repeat-violation rate, and generalization is unmeasured.
- Concurrent-trap semantics and relaxed-trap-ordering are unspecified (round3 lines 53-54), so runtime lesson attribution — the dynamic half of any loop-teaching story — is currently undefined.
- Lesson persistence (whether a cited rule stays learned within and across sessions, including compaction events) and repair-transcript context-cannibalization are both unmeasured, in opposite directions for the two principal positions.
