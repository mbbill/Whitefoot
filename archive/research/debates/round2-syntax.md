# Round 2 — Human-oriented syntax (`syntax`)

**Verdict:** `replace_with_alternative`

## Rationale

The prosecution wins on the core and the defense conceded it: under the corpus standing rule (caveats.md — cite exact cards or it does not ship) and the null hypothesis, human-oriented syntax holds an empty evidence hand. All confirmed enablers are semantic (F001–F009, N, A, D, C, J); syntax was explicitly deferred by the corpus (source-to-ir-lowering-sketches.md, caveats.md, A001 negative scope). On verification — the axis this project weights heaviest — sugar and elision are pure liability: each rule adds unverified frontend transformation surface at the exact soundness chokepoint the corpus flags (F003 negative scope, 'unsound frontend metadata emission' in open-questions.md, round1-index's lowering-survival gap), while N006's Alive2 discipline prices every rewrite at proof-grade cost and F006 (SPARK) is precedent that verification-first languages restrict surface expressiveness. This is replace_with_alternative rather than cut_from_core because the canonical form is not a passive default — it becomes a load-bearing requirement: the prosecution's benefit column (drift detection, node-path diagnostics, content-addressed caching, single-spelling dedup) only exists if the language mandates and polices those properties, so they are spelled out as obligations. Why not needs_evidence: the genuinely open empirical questions (token-cost ratio with real tokenizers — the 1.35–1.48x proxy reproduced in this hearing; transcription-distance vs precedence-misbinding error rates) govern only the defender's narrow write-path infix codec, and that codec is additive tooling over a fixed canonical core — a bijective renderer can be admitted later behind the defender's own A/B experiment without reopening the core decision. The core ruling is robust to either experimental outcome, since even the defender's proposal stores and echoes only canonical form. Deciding the core now is therefore safe and decisive; the codec stays out until evidence exists. All AI-writer behavioral claims remain EMPIRICAL-UNTESTED and are gated accordingly.

## Optimizer role

None carded. Every confirmed optimizer enabler in the corpus is semantic — aliasing/non-interference (F001, F002, F003), ownership (F006), numeric modes (N002), specialization/compile-time channels (D001, D004), memory/layout facts (A001) — and no card attributes any performance benefit to infix, precedence, sugar, or elision. Surface notation is erased at parse; both grammars yield the identical checked tree, so cutting notation forfeits zero optimization power. The corpus explicitly deferred syntax (source-to-ir-lowering-sketches.md: 'No syntax decisions yet'; caveats.md: scaffolding must not decide source syntax; A001 negative scope: 'Does not decide source syntax'). If anything, the corpus's practice points toward the alternative: N006 shows optimizer infrastructure already operates on single canonical forms, and F009's lesson is that performance-critical structure should be visible to tools rather than buried in implicit forms.

## Verification role

Negative for the feature, positive for the alternative — and verification is the decisive axis here. Every desugaring, precedence-resolution, and elision-inference rule is frontend transformation surface whose semantics-preservation must be proven and maintained; F003's negative scope (facts sound only if frontend obligations are true and survive lowering), the misleading-comparisons entry ('IR metadata solves source design' is misleading because metadata is sound only if frontend semantics are sound), open-questions.md's historical-failure list ('unsound frontend metadata emission'), and round1-index.md's cross-topic gap (no prototype validation that emitted facts survive lowering) all mark the frontend as the pipeline's soundness chokepoint. N006's Alive2 discipline implies each desugaring rewrite demands proof-grade validation; a zero-sugar core incurs zero such obligations. F006 (SPARK) is direct precedent for restricting surface expressiveness to keep the source-to-logic mapping simple. Additionally, elision of contract-bearing facts (aliasing/ownership per F001/F003, numeric modes per N002, effects, boundary types) would either destroy the explicit contract channel or force unverified re-inference — forbidden under the writer-fallibility premise. The canonical alternative makes the stored artifact identical to the checked tree and enables a checksum channel: writer-stated inferable facts cross-checked against inference, mismatch = hard error.

## AI-writer considerations

All AI-writer behavioral claims on both sides are EMPIRICAL-UNTESTED — the corpus has zero AI-codegen evidence cards, and the null-hypothesis win on syntax is procedural (syntax was out of scope by selection), not empirical. Verified in this hearing: the defender's token-cost proxy reproduces (S-expression forms cost 1.35–1.48x proxy tokens and 1.93x characters vs infix on six expression-dense kernel fragments; script at /private/tmp/claude-501/-Users-bytedance-Dev-xlang/035e62ef-9ecd-42d5-b288-f6f92231c257/scratchpad/tokcount.py) — but this is a regex proxy on fragments, not real BPE on whole programs, and declaration/control-flow-heavy code will inflate less. The two load-bearing untested hypotheses cut in opposite directions: (a) transcription-distance errors — converting infix spec formulas to prefix form induces silent operand-order/associativity errors that type-check and are invisible to the verifier; (b) precedence misbinding — writers trained on C-family corpora silently misparse their own infix under a misremembered precedence table, an error class canonical form eliminates by construction. Neither rate is measured; the codec question hinges on their difference. Uncontested machine-writer wins for canonical form: byte-detectable consistency drift across near-duplicates, deterministic O(n) parsing with local error reporting, node-path machine-readable diagnostics, content-addressed caching in the iteration loop (compile-time is a carded cost channel, D001/D004). Training-distribution penalty for an unfamiliar canonical form is an admissible, unquantified cost.

## What must be true

- The language defines exactly one canonical surface form: one spelling per semantic construct, one legal byte-level formatting, grammar tree isomorphic to the checked core representation; non-canonical input is a hard compiler error, not auto-formatted.
- Consistency-checking of repeated code is a first-class toolchain obligation, not a passive benefit: content-addressed subtree hashing, duplicate detection, and drift reports across near-identical AI-generated blocks must be emitted by the compiler, since byte-canonical form only pays off if the toolchain actually surfaces drift.
- Contract-bearing facts are never elidable in the stored artifact: aliasing/ownership (F001/F003), numeric mode selection (N002 — each arithmetic node names exactly one mode), effects, and types at function/module boundaries are always explicit; any compiler-inferred fact is immediately re-elaborated into the stored canonical form.
- Writer-stated inferable facts are legal and cross-checked against inference with mismatch as a hard error (redundancy as hallucination checksum), replacing the verification value that elision would have silently discarded.
- Diagnostics are machine-readable node paths into the canonical artifact the writer emitted, with deterministic local parse-error recovery — no resugaring machinery, no heuristic cascade errors.
- Parse/typecheck/verification results are content-addressed and cached on canonical bytes, converting the surface decision into iteration-loop latency wins (the carded compile-time channel, D001/D004) that offset the measured token-verbosity cost.
- The canonical grammar is designed to minimize gratuitous verbosity without adding alternate spellings (short fixed operator atoms, flat n-ary nodes for provably associative same-mode operations), directly attacking the one measured cost of the cut.
- The cut is policed at the toolchain boundary: no pretty-printer, macro layer, or out-of-core dialect may introduce a second stored spelling; any future write-path render codec must be bijective, version-pinned, enforce a parse-print-parse fixpoint invariant, echo only canonical form in all compiler output, and is admissible only after the gating A/B experiment succeeds.
- Local type inference and similar elision questions that interact with the type-system debates (checked ownership, static specialization) are explicitly not pre-decided by this ruling; whatever those debates admit must still re-elaborate into the fully explicit stored artifact.

## Follow-ups

- Run the gating A/B codegen experiment before admitting any infix write-path codec: identical kernel specs rendered to both grammars; measure generation tokens (real model tokenizers, whole programs, not the fragment proxy), first-parse success rate, silent semantic-error rate via differential testing against reference implementations (specifically operand-order/associativity swaps vs precedence misbindings), and repair iterations to green.
- Prototype the canonical grammar with node-path diagnostics and subtree hashing; measure parse/elaboration share of end-to-end compile latency and the realized content-addressed caching benefit in a simulated AI repair loop (D001/D004 channel, currently frontend-unquantified).
- Test the drift hypothesis: whether byte-canonical single-spelling code measurably reduces consistency drift across AI-generated near-duplicate blocks, and whether drift scales with per-copy verbosity.
- Measure the training-distribution penalty: initial defect rates of current models emitting the canonical form vs infix, and how fast the gap closes with in-context examples — this bounds the transition cost of the ruling.
- Open a syntax/AI-codegen evidence lane in the corpus (new card IDs) — every behavioral claim in this debate is currently uncardable, and the null-hypothesis win is procedural, not empirical.
- Write the toolchain-boundary policy document: allowed render layers, mandatory parse-print-parse fixpoint invariant, codec version pinning, and canonical-only storage/echo rules, so the cut cannot silently un-cut itself.
- Coordinate with the type-system debates (aliasing-model, dispatch-generics, numeric-semantics) on which elisions are actually inference questions owned by those topics, keeping the invariant that stored artifacts are fully elaborated.
