# Round 4 (v2) — Teachability synthesis under corrected D2

Corrected D2: spec compactness is the constraint; program verbosity acceptable (owner ruling). All three topics ended research_needed with strong convergent leading candidates.

## Spec architecture

- SINGLE MACHINE-CHECKED SPEC ARTIFACT (monolith-by-default): the language description — grammar, rules, annotation vocabulary, worked examples — lives in ONE normative, version-pinned, CI-checked artifact. This is the convergent core all three spec-budget positions share once overclaims are stripped: P3's regularity invariants enforced mechanically over the artifact, plus P1's procedural apparatus (counted token AND rule-count deltas on every design change, spec-delta column in decision-gates.md, kernel reserve priced first). 'Hard-in-procedure, number-by-measurement': no numeric token cap has cut authority until the calibration audit and spec-size ablation locate a real degradation knee — pre-registering 10-20k now would engrave an uncarded model-behavior claim (caveats.md standing rule; zero in-context-acquisition cards exist).
- KERNEL/LAYERS (P2) IS DEMOTED TO AN EXPERIMENT ARM, NOT ARCHITECTURE: layering grows TOTAL normative mass (extension law, layer registry, provisioning protocol) to shrink resident share, which is rational only if unrun experiments (U1/U7, context-competition, retrieval-substitution) hold; its partition also collides with decided floors (storage/lifetime contracts are mandatory on EVERY allocation per round2-synthesis line 29 — majority-path, not layer material; typed operation tables are core dispatch per line 27). RETAINED from P2 regardless: the conservative-extension law (any future layer only adds vocabulary at kernel extension points; unknown constructs are deterministic hard errors under canonical-form + closed-world), content-addressed version-pinned spec artifacts, and spec provisioning recorded in provenance.
- TEACHING PACK AS GENERATED BUILD ARTIFACT (P3's mechanism under spec-primary allocation): the spec + canonical worked examples + the diagnostics/report vocabulary are generated as a deterministic, provenance-checked, content-addressed pack — the round-2 consistency machinery (mechanism 2) applied to pedagogy. Example extraction is drift-free by construction under the single-spelling floor (extracted examples are re-verified subtrees); pack prose can still rot, so pack-faithfulness validation is a named open deliverable. The pack's writability CI gate runs ADVISORY only until the benchmark-transfer validity study passes — gating language evolution on current-model pass rates is the process-relative acceptance round-3 invariant 4 resists.
- SPEC-PRIMARY PEDAGOGY, LOOP AS REINFORCEMENT: the compact kernel spec carries the teaching burden; the repair loop carries prior-correction, mechanical edit-list execution, telemetry, and the conceded nonlocal-ownership residue. Every diagnostic is deterministic, byte-stable, one-pass-complete, and rule-cited pointing back INTO the spec (reinforcement, never a substitute lazy curriculum — lazy delivery would remove the cost pressure that enforces the rule cap, and a citation database of 'rule elaborations' would itself be a D2 violation wherever the text lives). Checker-invisible quality classes (region pessimization, context-dropping Result chains, operand swaps, performance-wrong code — D0's actual objective) route to by-construction design + differential/silent-wrong harness + lint channels, never to either teaching channel.
- NO-SHADOW-SPEC RULE (perimeter of the counted artifact): ALL normative content the writer must learn in-context counts against the one artifact — explicitly including the machine-readable diagnostics vocabulary, provenance-record schema, site-enumerated edit-list schema, and elaborated re-read view format (round2-synthesis Additions, lines 66-76). These are currently uncounted shadow spec in every proposed regime; corrected D2 makes them first-class budget items. One unified telemetry + gated spec-evolution channel (merging P1 distillation gate / P2 fleet ledger / P3 pack regeneration) — three parallel channels would reproduce the near-duplicate-machinery drift the charter forbids.
- SURFACE UNDER THIS ARCHITECTURE (leading candidate from the familiarity ruling): the round-2 alien-regular canonical grammar stands untouched (uniform node shape; Rust-shaped node inventory stays out — it adds productions, exactly what corrected D2 charges); lexicon may be selectively borrowed from the Rust prior only after a divergence census, with fully self-contained definitions (familiar names as mnemonics, no definition-by-delta, minted tokens at every divergence point such as unchecked-UB vs nsw/nuw-poison). Lexicon is first-order spec-budget-neutral; grammar shape is not. The parked bijective write-codec keeps familiarity purchasable later as tooling at zero spec cost — the reversibility asymmetry that de-risks keeping the canonical core alien.

## Regularity rules for the spec (machine-checkable invariants)

- ONE SPELLING PER CONSTRUCT: exactly one canonical spelling per semantic construct, one legal byte-level formatting, grammar isomorphic to the checked core tree; non-canonical input is a hard error, never auto-formatted (round-2 floor, now also the D2 invariant). Machine check: grammar has no alternative productions for the same node kind; no pretty-printer or codec introduces a second stored spelling.
- ZERO CONTEXT-DEPENDENT RULES: no rule whose meaning, applicability, or elaboration depends on surrounding context (no elision restored from context, no defaulting, no positionally-resolved meaning). Machine check: every production and typing rule is locally decidable from the node and its declared annotations.
- EMPTY EXCEPTION LISTS: no rule carries an exceptions clause; a needed exception is evidence the rule is wrong and must be replaced by one uniform rule (composition-over-addition forced structurally). Machine check: exception-clause count = 0 in the checked spec artifact.
- EACH FACT STATED ONCE: no normative fact appears twice in the artifact; cross-references, not restatement. Machine check: near-duplicate detection over normative clauses; duplication fails spec CI.
- NO SUGAR / NO ALTERNATE SURFACE / NO INFERENCE IN THE COUNTED SURFACE: syntactic sugar, alternate spellings, precedence tables, inference rules, and context-dependent elision are the enumerated budget-eaters (user-directives line 18) and are banned from the writable surface; the D2-cheap alternative — one uniform verbose rule — is always preferred.
- SPEC-REGRESSION RATCHET on rule count: rule count may not increase without a gate decision recording the counted delta and what was deleted or unified in exchange; caveat carried forward — the ratchet binds deltas from an unmeasured level, so it cannot certify the level itself (that is the calibration audit's job).
- CANONICAL RULE-INDIVIDUATION PROCEDURE (owner-defined, prerequisite): how schemas, parameterized rule families, and table rows count as rules — without it the ratchet is mega-rule-gameable (one giant uniform table gaming the count) and every gate's arithmetic is contestable.
- ONE POLICY, STATED ONCE, WITH A NAMED OWNER for each cross-cutting vocabulary decision: e.g., the borrow-vs-mint lexicon policy, the Result-vs-trap classification rule, annotation spelling canon — a policy family is one rule, not N ad-hoc judgments (guards against the prior-alignment invariant degenerating into a special-case generator).
- NO-SHADOW-SPEC: all normative content the writer needs in-context is inside the counted artifact — including diagnostics vocabulary, provenance-record schema, edit-list schema, re-read view format. Nothing normative lives only in toolchain code, layer docs, or a citation database.
- CLOSED DIAGNOSTIC VOCABULARY BOUND TO RULES: every checker rejection cites a stable rule ID whose canonical text is in the spec; rule-to-emitter coverage audit guarantees the bijection where one exists; the sound-but-rejected restructure-demand class (soundness-over-completeness lever, decision-gates line 74) gets its own designed diagnostic class rather than free text.
- CONSERVATIVE-EXTENSION LAW: any future spec partition or layer may only ADD vocabulary at kernel extension points; unknown constructs are deterministic hard errors (safe under canonical-form + closed-world: under-provisioning costs iterations, never soundness).
- UNCOUNTED-LEDGER POLICING: a registry and periodic audit of toolchain-side normative mass, with consolidation authority over duplicated drift machinery — every accounting regime relocates mass to its uncounted side, and the corpus's only documented accretion failure happened exactly there.

## Open tensions

- D2's UNIT OF MEASURE is unruled at the decisive margin (the single item that determines whose arithmetic is admissible): rule-count/irregularity-only (line 18 reading) vs total spec+examples in-context footprint (line 16 places examples inside the description). A rule-count-only ruling decides regularity-first by arithmetic; total-footprint makes the model experiments decisive. Sub-question with the same force for pedagogy: do diagnostic-delivered rule elaborations, witnesses, and anti-prior notes count against the budget or as toolchain output? Both are owner rulings, not experiments — and both are still pending.
- TOTAL-DESCRIPTION vs RESIDENT-CONTEXT accounting: D2's text binds total description; its stated rationale (in-context teaching) is a residency argument; the spec shares the window with the mandatory re-read artifact, edit lists, diagnostics, and the program under repair. Monolith-with-cap and kernel-plus-layers are each coherent only under one reading; adopting either as permanent architecture pre-empts the arbitration experiments all sides defer to. Current stance (monolith default, P2 as experiment arm) is a holding position, not a resolution.
- RULE INDIVIDUATION IS UNDEFINED AND GAMEABLE: mega-rules, parameterized schemas, and giant uniform tables can launder irregularity through the counter; conversely token pressure Goodharts toward deleting exactly what D2 values (worked examples, fully-spelled quantifiers) and compressing explicit statements into denser context-dependent forms. Neither tokens nor rules suffices alone; the canonical counting procedure exists as a requirement but not yet as a document.
- EVERY REGIME RELOCATES MASS TO ITS UNCOUNTED SIDE (P1→toolchain, P2→layer specs, P3→gate procedures and checked-spec tooling) and the corpus's only documented accretion failure occurred on the uncounted side; the audit registry is adopted in principle but no one holds cut authority over uncounted complexity — the gate is one-sided by construction until someone does.
- SPEC-COST AS TIE-BREAKER vs DECIDED EXPERIMENT PROCEDURE: corrected D2 gives the two needs_evidence topics (interior annotations, interfaces/bounded quantification) a measurable criterion NOW, and the pressure to use it will recur; the tie-breaker discipline (pre-registered input only, recorded reopening conditions) is a fence, and the interior-annotation case shows how hard it leans — D2 stacks a real handicap on the elision arm before its experiment runs.
- KERNEL RESERVE IS UNKNOWN AND MOVING: the FFI wall (zero primary-source cards), concurrency surface (C004 kernel undesigned; concurrent-trap semantics unspecified, round-3 lines 53-54), region/arena contracts, and both needs_evidence outcomes will add counted mass after any regime hardens; D1a's checker-simplification levers plausibly SHRINK the ownership spec but are also unpriced. A budget could be 'set' and then discovered already spent.
- MANDATED SPEC-HEAVY FLOORS vs FEW-RULES ASPIRATION: round 2's writer-visible machinery (operation-table coherence, contract groups, per-allocation storage contracts, capability predicates, mode vocabulary) is law and is not free; if the calibration audit prices the floors near any plausible knee, D2 pressure lands on the only compressible remainder — examples and diagnostics vocabulary — which is exactly the Goodhart failure the ruling warns against.
- ELABORATOR-AS-CODEC / SOURCE-OF-RECORD (round2-synthesis line 113, unresolved and now D2-loaded): if interior elision wins its experiment, writer-emitted and stored surfaces diverge, the elaborator becomes a codec under the bijectivity/fixpoint rules, and no verdict says which artifact is source of record during repair loops; familiarity and regularity could split across the write and re-read surfaces — all surface positions assumed one surface serves both.
- FALSIFIABILITY ASYMMETRY OF FAMILIARITY: with a familiar lexicon, a passing teachability eval cannot distinguish spec-driven from prior-driven competence; spec bugs masked by prior compensation surface only when the prior shifts. The alien-rendering validation requirement mitigates but does not price this long-term assurance cost.
- PROCESS-RELATIVE ACCEPTANCE vs ARTIFACT-CHECKABLE COMPETENCE: any CI gate scored on current-model pass rates (P3's pack gate; any hardened knee threshold) re-instantiates the process-relativity round-3 invariant 4 resists and risks baking 2026 model idioms into permanent language design; the advisory-until-transfer-validated compromise plus per-generation re-validation is a mitigation with an unresolved endpoint — measured numbers have a shelf life shorter than the irreversibility of the decisions they settle.
- ZERO IN-CONTEXT-ACQUISITION CARDS REMAIN THE ROOT CONDITION: every load-bearing learnability claim in all three rulings (degradation knee, irregularity dominance, eager-vs-lazy economics, penalty decay, examples exchange rate) is EMPIRICAL-UNTESTED; the ordered experiment lane is still unopened, and until it opens, everything above 'decided now' is governance scaffolding around an unmeasured phenomenon.
