## Rationale

The debate's decidable core and its empirical core separate cleanly, and the critic's analysis shows all three positions already agree on the decidable core once defects are removed. Decidable now: D2 must be operationalized as a gate, not advocacy — the corpus's own record (periphery accretion and six uncoordinated drift mechanisms produced UNDER the existing cut discipline) refutes P3's claim that the forcing function is already satisfied, while caveats.md's standing rule and the zero-card state of in-context acquisition refute P1's authority to pre-register 10-20k as anything but a placeholder (its own concession: hard-in-procedure, not hard-in-threshold). The intersection — checked spec artifact, style-invariant regularity invariants, counted token+rule deltas, kernel-reserve-first sequencing, no-shadow-spec — is supported by decided law and carded evidence on the non-model axis (F003: every writable rule is frontend soundness surface; N006: per-rule proof-grade validation cost; F006: restriction as the verification-first route) and requires no model experiment to justify. Genuinely empirical and therefore not decidable today: (a) whether a token-count degradation knee exists and where (P1's number), (b) whether irregularity dominates raw size (P3's causal claim and D2's own stated mechanism), and (c) whether total description or per-task resident share is the binding variable (the P1-vs-P2 architecture fork — the critic's sharpest tension). Engraving any of these as architecture before the ordered experiments run would repeat the exact failure mode this project's gate methodology exists to prevent: a design becoming law because it is governance-attractive rather than carded. P2's architecture is additionally ruled premature on non-empirical grounds — its partition contradicts decided floors (storage contracts are every-allocation mandatory; operation tables are core), and its load-bearing citations (F001 verification status, D004, D009, N001) fail verification. Sequencing note: the D1a/D0a rulings have now landed, so the calibration audit is unblocked and should price the SIMPLIFIED checker calculus D1a mandates, not rustc-class rules.

## Position assessments (post-critique)

### P1 — Hard CI-enforced whole-spec budget (10-20k token anchor, kernel reserve priced first, calibration audit before cut authority)

**Strengths:** Correctly identifies the governance gap: D2 names spec description-length/regularity the first-class metric yet nothing enforces it, and the corpus's own recorded failure (round2-synthesis Open tensions: periphery accreting into 'a second, evidence-free language', six uncoordinated drift mechanisms) shows the existing null-hypothesis-CUT discipline did not bound uncounted growth. Its procedural apparatus survives the critique intact: counted token+rule deltas on every design change, whole-spec DIFFERENTIAL accounting (a construct that deletes N ad-hoc rules is correctly kept), kernel-reserve-priced-first sequencing, the no-shadow-spec rule, and spec-as-CI-artifact — which is also the missing prerequisite for the syntax A/B, drift, and every in-context experiment round 2 already ordered. The dual-support argument partially survives: a smaller writable surface genuinely shrinks the F003 frontend soundness chokepoint and the N006-priced validation program even if token count is the wrong unit. Its concessions are honest and mostly correct.

**Weaknesses:** The headline number does not survive its own concessions: the 10-20k anchor is uncarded (no one has tokenized Lua 5.4, R7RS-small, or the drafted floors with a real BPE tokenizer), the position admits full manuals plausibly bust 20k, and its own risk analysis (a moved number permanently damages credibility) argues against pre-registering a number it expects to move. The retrodiction claim ('the budget reproduces the round-2 verdict lattice') was never computed and is one-sided — round 2 also MANDATED spec-heavy writer-visible machinery (typed operation tables with coherence rules, contract-group kernel, FFI declaration frames, named mode vocabulary, env-struct diagnostics). Card overreach confirmed by the critic: C004's confirmed scope is data-race prevention only (07-02 amendment sharpened it), not evidence the full D1 kernel (ownership/borrow system, obligation ledger, FFI frames) fits any budget; N006 prices optimizer folds, not spec tokens — the carded benefit maps to rule/production count, collapsing most of the carded case into P3's regime. Internal contradiction between 'spec-cost as the tie-breaker NOW' for the open tensions and concession 5's 'pre-registered input to experiments only' is unresolved. What survives is 'hard-in-procedure, not hard-in-threshold' — operationally near-identical to P3 plus a premature constant.

### P2 — Kernel-scoped hard budget + on-demand layered specs (conservative-extension law, census-gated 80% kernel path)

**Strengths:** Names the real deployment pressure the others under-weight: the spec shares context with the mandatory re-read artifact, site-enumerated edit lists, machine-readable diagnostics, and the program under repair (round2-synthesis 'Additions' section), so resident share is bounded whether designed or discovered. Several components have standalone value regardless of layering: the conservative-extension law (layers only add vocabulary at kernel extension points; unknown constructs are deterministic hard errors under canonical-form + closed-world, so under-provisioning costs iterations, never soundness), spec provisioning recorded in provenance, content-addressed version-pinned spec artifacts, and the observation that round-3's overflow ruling (wrap fallback legal) supplies a kernel-safe default arithmetic. Its U1-U8 empirical program is the best-specified of the three and should largely be adopted as the experiment battery.

**Weaknesses:** The architecture answers resident-context share while D2's text constrains TOTAL description — layering GROWS total normative mass (extension law, layer registry, provisioning protocol, per-layer budgets, version-pinning) to shrink residency, a trade rational only if its own untested premises U1/U7 hold; it concedes it abandons a hard bound on total spec size. The kernel/layer partition collides with decided floors at named points: storage/lifetime contracts are mandatory on EVERY allocation (round2-synthesis line 29) — majority-path surface, not layer material — and typed operation tables are the blessed CORE dispatch form (line 27). Citation defects verified against the corpus: 'F001 CONFIRMED 2026-07-02' is false (the 07-02 set is A001/A002/A006/C004/D001/D002/D003/D005/D007/J004); D004 says nothing about spec serving; D009 is provisional and documents performance degradation, not silent semantic reinterpretation; the N001 flagship example was never writable-subset mass (the UB/poison ladder is compiler-internal under either architecture). The 80%-kernel claim rests on a census (U4) that does not exist, and layer-membership knowledge is itself uncounted description mass of exactly the context-dependent kind D2 taxes. Premature as adopted architecture; valuable as an experiment arm.

### P3 — No pre-registered number: machine-checkable regularity invariants (one spelling, zero context-dependence, empty exception lists, stated-once) + ratcheted rule count; numeric budget only measurement-derived

**Strengths:** Most faithful operationalization of corrected D2's literal text — the invariants map one-to-one onto the directive's enumerated budget-eaters (sugar, alternate spellings, inference, elision, special cases) and are style- and tokenizer-invariant, avoiding re-enshrining the exact proxy failure the corpus already recorded (round2-syntax's 1.35-1.48x regex-proxy lesson). Matches the project's demonstrated decision procedure (measure-then-threshold with pre-committed gates, per the round-2 verdicts) and caveats.md's standing rule: with zero in-context-acquisition cards, any specific number silently engraves an untested model-behavior claim as law. Its Goodhart analysis of a binding cap is credible and specific: the cheapest tokens to cut are exactly what D2 values (worked examples, fully-spelled quantifiers), and compression pressure converts explicit statements into denser context-dependent formulations — irregularity increased while the metric improves. The spec-as-checked-artifact and spec-regression-ratchet mechanics are adoptable immediately and force composition-over-addition, which is D2's uniformity demand made structural.

**Weaknesses:** The ratchet inherits the arbitrariness it condemns one level down: initial per-layer rule bounds are judgment numbers with zero acquisition data, rule individuation is undefined and gameable (mega-rules, parameterized schemas, giant uniform tables), and a ratchet binds the delta from an unmeasured level — it cannot detect that the level is already too high. Governance asymmetry cuts against it: caps fail loudly, ratchet gates fail quietly, and its learning-curve harness is a critical-path dependency whose delay forces stalled evolution or a waived gate (its own words: worse than a crude cap). Its 'forcing function already satisfied' claim is contradicted by the corpus it cites — periphery accretion happened UNDER the existing discipline. It commits the offense it polices: the ~2x prose-variance figure is invented, and D001 is cited for a procedural claim that belongs to the generics verdict, not the card. Its style-invariance superiority claim holds for the three binary invariants but not for the quantitative part ('few', the ratchet) that actually replaces the cap.

## Key tradeoffs

- Pre-registered number vs measured-later number: a hard constant is cheap to adjudicate, fails loudly, and forces real prioritization fights now — but with zero acquisition cards it engraves an untested model-behavior claim, invites Goodhart compression (example deletion, rule-merging into context-dependence), and a later recalibration permanently damages the gate's 'hard' credibility; a measured-later number is evidence-honest but fails quietly and stalls if the learning-curve harness slips.
- Total-description vs resident-context accounting — the sharpest tension: a monolith-with-cap is coherent only if total description binds; kernel-plus-retrieved-layers is coherent only if resident share binds (it grows total description to shrink residency). D2's text says total; its stated rationale (in-context teaching) is a residency argument; the arbitrating experiments (spec-size ablation, context-competition, retrieval-substitution) are unrun. Adopting either architecture now pre-empts the experiment all sides claim to defer to.
- Token count vs rule/production count as the gated unit: tokens capture everything D2 names (including examples) but are tokenizer-dependent and prose-gameable; rule counts are style-invariant and map to the carded costs (F003 frontend soundness surface, N006 proof-grade validation per rule) but lack an individuation procedure and are mega-rule-gameable. Neither suffices alone; both need a canonical counting procedure that no position defines.
- Counted-ledger discipline vs uncounted-ledger accretion: every regime relocates mass to its uncounted side (P1 to toolchain, P2 to layer specs, P3 to gate procedures and checked-spec tooling), yet the corpus's only documented accretion failure occurred precisely on the uncounted side; registries are proposed but no position grants cut authority over uncounted complexity, making all three one-sided by construction.
- Rules vs examples in one budget pool: rules interact combinatorially (P1's own interaction-complexity concession) while worked examples are near-additive and explicitly part of D2's teaching channel; a single budget over heterogeneous pedagogical material risks optimizing away the cheaper-per-token teaching medium — the exchange rate is untested (examples-ratio ablation).
- Budget as tie-breaker for the open tensions vs experiment deferral: spec-cost is the only measurable criterion available NOW for interfaces/closures/sharing (P1's strongest practical pitch), but round 2 ruled those 'empirically unarbitrable today' with ordered experiments and pre-committed thresholds; a budget-broken tie that pre-empts a needs_evidence verdict must carry a recorded reopening condition or it silently overrides decided procedure.

## Evidence gaps

- Zero AI-codegen / in-context-acquisition evidence cards exist (round2-syntax records this explicitly and orders a new lane that is still unopened). Every load-bearing learnability claim — spec-size degradation knee, irregularity-vs-size dominance, context competition, retrieval substitution, rules-vs-examples exchange rate, cross-model transfer — is EMPIRICAL-UNTESTED.
- No calibration measurements exist: Lua 5.4, R7RS-small, and (critically) a drafted round-2-floors + round-3-envelope spec have never been priced in real BPE tokens or counted rules. P1's 10-20k anchor, P1's retrodiction claim, and P2's 80%-kernel partition are all unpriced assertions; the floor-pricing audit is the cheapest missing artifact and blocks all three positions.
- Kernel-reserve size is unknown and moving: the FFI wall (research_needed, zero primary-source cards), concurrency surface (C004 kernel undesigned), region/arena contracts, and the two needs_evidence outcomes (interior annotations, interfaces/bounded quantification) will add spec mass after any regime hardens. Note the D1a ruling's checker-simplification levers (explicit regions/borrows over inference, lexical borrows, minimal Featherweight-Rust/Oxide-class calculus) plausibly SHRINK the ownership spec description — also unpriced.
- No canonical counting procedure is defined anywhere: which tokenizer(s), how rules are individuated (schema vs instantiations, table rows, quantified rule families), and what counts as normative. Without it every gate's arithmetic is contestable regardless of which metric wins — this is a procedural/owner decision, not an experiment.
- Writer-emitted artifact schemas are uncounted shadow spec in all three regimes: the writer must learn in-context to emit provenance records (itself flagged EMPIRICAL-UNTESTED at round2-synthesis line 76) and to consume edit lists, elaborated re-read views, and machine-readable diagnostics — description mass excluded from P1's writable subset and P2's kernel alike.
- The U4 workload census corpus (share of functions needing only kernel constructs in representative AI-generated performance-critical code) does not exist; P2's central architectural claim is unfalsifiable until it does.
- D009 remains an unverified provisional card (decision-gates line 49) yet is used load-bearing by both P1 (pricing context-dependence) and P2 (conservative-extension safety analogy); its claim is Julia performance degradation, and the semantic-reinterpretation reading is an uncarded extrapolation.
