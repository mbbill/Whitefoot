# Round 2 — Parametric generics (`generics`)

**Verdict:** `keep_essential`

## Rationale

The null hypothesis is that generics are not in the core, and the prosecution must show the named alternative carries the load. The ruling turns on a structural finding: the alternative does not instantiate the null hypothesis. To declare and CHECK that copies instantiate a common template, the language must contain a parameterized template sub-language, once-checked template well-typedness against bounds (or diagnostics fan out per-copy, post-expansion), and a computable instantiation relation copy == template[T]. A compiler that can compute template[T] to check a copy can emit template[T]; the expander is a strict subset of the checker. So the 'cut' keeps essentially all of generics' language complexity, discards only the deterministic expansion step — the one part machines do perfectly — and adds back: O(instantiation-closure) token mass, regeneration cascades on template edits, writer-side whole-program closure computation (D001/D005/D010 show compilers do this on demand), N-fold diagnostic fan-out, and a new drift channel (checker acceptance bugs ship undetected wrong programs — the prosecution's 'detected-late drift, never miscompilation' claim is wrong on inspection, since an admitted divergent copy is compiled faithfully into a semantically wrong artifact). The prosecution's own risk list concedes the two remaining holes: 'mechanism regrowth' into a parametric type system living inside the checker, and checked-divergence escape hatches that reinstate exactly the unchecked near-duplicate regions the CHECK exists to eliminate. The defender's dilemma stands: full substitution-equality makes every copy a redundant echo of compiler output; anything weaker institutionalizes an unverifiable drift surface in a charter where verification matters more, not less. What the prosecution genuinely proved is granted and shapes the form of the keep: D001 means generics add no per-instance optimizer information, so the case for compiler-side expansion rests on cross-copy identity, deferral of the specialize/share decision (D004, D005, F009 — the axis round-1 explicitly held open pending measurement, which source-level pre-expansion would foreclose irreversibly), demand-driven instantiation throttling D009-class over-specialization, and check-once/instantiate-many verification with F003 obligations transferring by substitution. The prosecution's legitimate fears — inference/solving subsystems, F009-style metastasis into IR, witness-table ABI, the F003 shared-body question — are severable, and the ruling severs them: the kept core is monomorphization-only with explicit instantiation, which deletes that open-question cluster from the shipped core as effectively as cutting would, without paying the alternative's costs. J001/J002 settle the endpoint question in the keep's favor: every performance strategy converges on concrete-type specialization, the information demand is constant and only the channel differs, and a bounds-checked template with compiler expansion is the cheapest channel with zero guard/deopt machinery and zero redundant source. Why keep_essential rather than needs_evidence: the missing empirics (drift rates, token magnitudes, mono-vs-shared bloat — all EMPIRICAL-UNTESTED, all D-cards provisional per caveats.md) arbitrate the eventual FORM of generics (mono-only vs bounded-sharing vs Swift-style hybrid), which round-1 already holds open behind a measurement program. They do not arbitrate the in/out decision, because the collapse argument is logical, not empirical: both parties' designs contain the template mechanism, so 'out' was never actually on offer — only 'in, plus a redundant serialized expansion with worse trust properties.' Verification necessity is thereby proven in the only sense available: the anti-drift check both sides require IS parametric generics, and generation-by-construction dominates check-after-generation. The verdict is decisive on that ground and honest about the rest: if adversarial verification breaks D001, or the AI-codegen experiments show template abstraction catastrophically degrades writer reliability, the question reopens — at the level of surface form, not of the mechanism's presence in the core.

## Optimizer role

Per-instance, generics add nothing: D001 attests monomorphization yields runtime behavior identical to hand-written concrete code, so the optimizer sees the same machine-code endpoint either way — the prosecution is granted this fully. The feature's real optimizer role is structural: (1) cross-instantiation identity, which enables cross-unit deduplication (D004 context; rustc share-generics is a named round-1 follow-up) and which textual near-duplicate copies destroy the moment any drift or checked-divergence hatch exists; (2) deferral of the specialize/share decision, keeping open the D005 bounded-sharing hybrid and the F009 Swift-SIL fourth model (IR-visible generic structure) that round-1 explicitly held open pending measurement — source-level pre-expansion forecloses that axis irreversibly, with no compiler knob to un-expand; (3) demand-driven instantiation (D001/D005/D010), which throttles the over-specialization compile-time/cache pressure documented in D009's negative scope; and (4) delivery of the concrete-type endpoint that J001/J002 show every high-performance runtime converges on, through a deterministic channel with zero guard/deopt/invalidation machinery. In the minimal kept form (frontend monomorphization only), the round-1 decisive open question — whether shared/erased bodies can discharge F003 noalias/alias.scope obligations — does not enter the shipped core: every instantiation discharges F003 per-instance exactly as concrete code does. That question attaches only to the sharing option, which stays gated behind the check round-1 already ordered.

## Verification role

This is the decisive axis, and it is where the alternative collapses. Both parties agree N specializations must be verifiable as instantiations of one declared intent — the alternative's own CHECK concedes drift is the expected failure mode of a fallible writer at scale. But any check adequate to that job IS parametric generics: it needs a template sub-language with type parameters, template well-typedness checked once against declared bounds (otherwise errors surface per-copy post-expansion, defeating machine-readable diagnostics), and a computable instantiation relation copy == template[T]. If the checker can compute template[T] to compare, the compiler can emit template[T] directly; the expander is a strict subset of the checker (substitution without tree-equality). Generation-by-construction is therefore strictly stronger than check-after-generation: there is no window where divergent copies exist, and no checker whose acceptance bugs silently ship semantically wrong programs. The prosecution's trust-path argument fails on inspection — a checker bug that admits a divergent copy is not 'detected-late drift'; it is an undetected wrong program, equivalent in running-program severity to a miscompile. Kept generics also give check-once/instantiate-many verification: a template checked against bounds is a universally quantified proof over all instantiations, and F003-class discharge obligations established once transfer by substitution soundness, whereas per-copy re-verification scales with the instantiation closure and any tractability weakening of the check reopens the drift surface the charter forbids.

## AI-writer considerations

All AI-writer behavioral claims on both sides are EMPIRICAL-UNTESTED — the corpus contains zero AI-codegen evidence. That said, the admissible cost directions are forced by construction and favor keep: (1) token/context mass scales with the transitive instantiation closure under written-out copies versus O(1) in instantiation count for a template — generated, stored, and re-read on every edit; (2) template edits under the alternative trigger regeneration cascades across every downstream module, each rewrite a fresh hallucination opportunity, with a stale-copy window until the build-gating check closes it; (3) deciding which copies must exist is a whole-program call-graph fixpoint (generic-calls-generic at caller parameters) that compilers perform internally on demand (D001/D005/D010) — the alternative shifts it onto the writer or external tooling; (4) one template-level defect fans out as N per-copy diagnostics unless tooling re-aggregates by template identity — machinery generics gets for free. Verbosity being admissible cuts FOR the minimal kept form: instantiation arguments can be fully explicit, deleting the inference/solving subsystem the prosecution rightly fears, at pure token cost the charter accepts. The one open writer-side risk on the keep side: whether AI writers handle template abstraction (binding, bounds, explicit instantiation) with lower error rates than near-duplicate emission — untested in both directions, and the experiment is named in follow-ups.

## What must be true

- The kept core is minimal: monomorphization-only parametric generics, expansion always compiler-side, explicit instantiation arguments (no global inference or bound solving in the trust path), polymorphic recursion and higher-rank parameters excluded so the instantiation closure is decidable and compile time bounded.
- Templates are type-checked once against declared bounds BEFORE instantiation (Rust-style, not C++-post-expansion-style), so diagnostics are template-level and machine-readable rather than fanning out per instance.
- Dynamic dispatch remains a separate, explicit, restricted construct (D002/D003) — round-1's leading-candidate architecture — and generic dispatchable methods stay excluded from dyn interfaces pending the escape-hatch design spike; the D003 composition hole is carried as a documented limitation, not papered over.
- No IR-level generic ABI, witness tables, or shared-body machinery enters the core now: the D005/F009 sharing-hybrid axis stays open exactly as round-1 gated it — behind the F003 shared-body discharge check and the mono-vs-shared-vs-dictionary measurement program with pre-committed thresholds.
- Legitimate per-type divergence (SIMD widths, layout-specific loop shapes) is expressed as explicit, individually-checked overloads/specializations visible in source — never as unchecked divergence from a template — with coherence rules making wrong-instantiation selection a hard resolution error.
- The load-bearing provisional cards D001, D002, D003, D005, D007 survive adversarial verification per the caveats.md standing rule; if D001's concrete-code-equivalence claim falls, this ruling is revisited.
- The AI-codegen experiment program runs and does not show that writers are dramatically more reliable emitting checked near-duplicate copies than instantiating templates; a strong contrary result reopens the question at the level of form, not principle, since the checked-copy design still contains the template mechanism.
